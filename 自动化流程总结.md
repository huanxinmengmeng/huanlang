# 幻语构建运行全链路自动化流程

## 1. 流程概述

本自动化流程实现了从幻语源代码到可执行程序运行的完整链路，包括源代码识别、构建配置、编译执行、结果验证等环节。流程通过 PowerShell 脚本实现，提供了详细的日志记录和错误处理机制，确保整个过程的可追溯性和可靠性。

## 2. 核心组件

### 2.1 自动化脚本

- **huan_pipeline.ps1**：主自动化脚本，实现完整的构建运行流程
- **huan_config.json**：配置文件模板，用于自定义构建和运行参数

### 2.2 文档

- **构建运行全链路自动化配置.md**：详细的配置文档

## 3. 功能特性

### 3.1 源代码识别
- 自动识别 `.hl` 和 `.幻` 后缀的源代码文件
- 支持单个文件和项目目录两种模式
- 验证文件存在性和类型正确性

### 3.2 构建配置优化
- 自动选择合适的 huan.exe 版本（发布版或调试版）
- 支持自定义构建参数（优化级别、目标平台、特性等）
- 智能默认配置，减少用户配置负担

### 3.3 错误捕获与处理
- 实时捕获构建和运行过程中的错误
- 详细的错误日志记录
- 明确的错误提示和问题定位

### 3.4 编译结果验证
- 验证构建是否成功完成
- 验证运行是否成功执行
- 保存运行输出供后续分析

### 3.5 运行环境配置
- 支持自定义环境变量
- 支持自定义运行参数
- 环境变量自动清理，避免污染

### 3.6 可追溯性
- 详细的日志记录（时间戳、操作、状态）
- 多级别日志（Error、Warning、Info、Verbose）
- 完整的执行过程记录

## 4. 使用方法

### 4.1 基本用法

```powershell
# 构建并运行单个文件
.\huan_pipeline.ps1 -SourcePath examples\basic\你好世界.hl

# 构建并运行项目目录
.\huan_pipeline.ps1 -SourcePath examples\basic_project

# 发布模式构建
.\huan_pipeline.ps1 -SourcePath examples\basic\你好世界.hl -Release

# 详细模式
.\huan_pipeline.ps1 -SourcePath examples\basic\你好世界.hl -Verbose

# 自定义输出目录
.\huan_pipeline.ps1 -SourcePath examples\basic\你好世界.hl -OutputPath my_output
```

### 4.2 高级配置

**自定义配置文件 `huan_config.json`**：

```json
{
  "build": {
    "optimize": "release",
    "target": "native",
    "features": ["llvm-backend"]
  },
  "run": {
    "args": ["--verbose"],
    "env": {
      "DEBUG": "true",
      "LOG_LEVEL": "info"
    }
  }
}
```

## 6. 日志系统

### 6.1 日志文件

| 文件名 | 路径 | 描述 |
|-------|------|------|
| pipeline.log | `output_dir/` | 主日志文件，包含所有操作信息 |
| error.log | `output_dir/` | 错误日志文件，仅包含错误信息 |
| run_output.txt | `output_dir/` | 程序运行输出 |

### 6.2 日志级别

- **Error**：错误信息（红色）
- **Warning**：警告信息（黄色）
- **Info**：普通信息（绿色）
- **Verbose**：详细信息（青色，仅 `-Verbose` 模式）

## 7. 错误处理

### 7.1 错误类型

| 错误类型 | 描述 | 处理方式 |
|---------|------|----------|
| 路径错误 | 源代码路径不存在 | 停止执行并提示错误 |
| 文件类型错误 | 不是幻语源代码文件 | 停止执行并提示错误 |
| 构建错误 | 构建过程失败 | 捕获错误信息并停止执行 |
| 运行错误 | 运行过程失败 | 捕获错误信息并停止执行 |
| 配置错误 | 配置文件无效 | 使用默认配置并警告 |

### 7.2 错误定位

当出现错误时，系统会：
1. 在控制台显示红色错误信息
2. 将详细错误信息写入 `error.log` 文件
3. 提供明确的错误原因和建议解决方案

## 8. 最佳实践

### 8.1 开发环境
- 使用默认配置（debug 模式）
- 启用 `-Verbose` 模式获取详细日志
- 定期清理输出目录

### 8.2 生产环境
- 使用 `-Release` 参数构建优化版本
- 合理设置环境变量和运行参数
- 保留日志文件用于问题排查

### 8.3 大型项目
- 使用项目目录模式，配合 `幻语包.toml` 配置
- 考虑使用工作区配置管理多包项目
- 利用特性标志控制功能开关

## 9. 性能优化

- 对于频繁构建的项目，使用 `target/release/huan.exe` 获得更好性能
- 对于大型项目，考虑使用增量构建
- 合理设置 `features` 以减少构建时间
- 使用 `-Release` 模式获得优化的可执行文件

## 10. 故障排查

当遇到问题时：
1. 检查 `error.log` 文件获取详细错误信息
2. 检查 `pipeline.log` 文件了解完整执行过程
3. 启用 `-Verbose` 模式获取更详细的日志
4. 验证 huan.exe 是否正确构建
5. 检查源代码文件是否存在且格式正确

本文档介绍幻语编程语言的调试工具和测试方法。

## 调试工具

### 命令行调试器

```bash
# 启动调试器
huanlang debug 文件名.hl

# 带断点启动
huanlang debug --break main 文件名.hl
```

### 调试命令

在调试器中，您可以使用以下命令：

| 命令 | 描述 |
|------|------|
| `run` | 运行程序 |
| `step` | 单步执行 |
| `next` | 跳过函数调用 |
| `continue` | 继续执行直到下一个断点 |
| `break` | 设置断点 |
| `print` | 打印变量值 |
| `backtrace` | 显示调用栈 |
| `quit` | 退出调试器 |

### 示例会话

```
$ huanlang debug example.hl
(hldbg) break main

(hldbg) run

(hldbg) step
5:     打印(y);
(hldbg) print y

(hldbg) continue

```


### 断言

```hl
导入 "标准库/测试" 作为 测试;

函数 主函数() {
    令 x = 10;
    
    // 断言 x 等于 10
    测试.断言.等于(x, 10, "x 应该等于 10");
    
    // 断言 x 大于 5
    测试.断言.大于(x, 5, "x 应该大于 5");
    
    // 断言条件为真
    测试.断言.为真(x > 0, "x 应该大于 0");
}
```

## 测试框架

### 测试类型

幻语支持以下类型的测试：

1. **单元测试**：测试单个函数或模块
2. **集成测试**：测试多个模块的交互
3. **端到端测试**：测试完整的应用流程
4. **属性测试**：测试函数的属性和不变量
5. **基准测试**：测试代码性能

### 编写单元测试

```hl
// tests/单元测试.hl
测试 测试加法() {
    函数 加法(a: 整数, b: 整数) -> 整数 {
        返回 a + b;
    }
    
    令 结果 = 加法(1, 2);
    断言.等于(结果, 3, "1 + 2 应该等于 3");
}

测试 测试减法() {
    函数 减法(a: 整数, b: 整数) -> 整数 {
        返回 a - b;
    }
    
    令 结果 = 减法(5, 3);
    断言.等于(结果, 2, "5 - 3 应该等于 2");
}
```

### 测试断言

| 断言方法 | 描述 |
|---------|------|
| `等于` | 检查两个值是否相等 |
| `不等于` | 检查两个值是否不相等 |
| `大于` | 检查第一个值是否大于第二个值 |
| `小于` | 检查第一个值是否小于第二个值 |
| `大于等于` | 检查第一个值是否大于等于第二个值 |
| `小于等于` | 检查第一个值是否小于等于第二个值 |
| `为真` | 检查值是否为真 |
| `为假` | 检查值是否为假 |
| `为无` | 检查值是否为无 |
| `为有` | 检查值是否为有 |

### 运行测试

```bash
# 运行所有测试
huanlang test

# 运行特定测试文件
huanlang test tests/单元测试.hl

# 运行特定测试
huanlang test 测试加法

# 运行测试并显示详细输出
huanlang test --verbose

# 运行测试并生成覆盖率报告
huanlang test --coverage
```

## 集成测试

### 编写集成测试

```hl
// tests/集成测试.hl
导入 "../src/lib" 作为 示例;

测试 测试模块集成() {
    令 结果 = 示例.加法(10, 20);
    断言.等于(结果, 30, "模块加法功能应该正常工作");
    
    令 消息 = 示例.问候("测试");
    断言.包含(消息, "测试", "问候消息应该包含名称");
}
```

### 测试环境设置

```hl
测试 测试需要环境变量() {
    // 设置测试环境
    环境.设置环境变量("TEST_VAR", "test_value");
    
    // 测试代码
    令 值 = 环境.获取环境变量("TEST_VAR");
    断言.等于(值, "test_value", "环境变量应该被正确设置");
}
```

## 属性测试

### 编写属性测试

```hl
// tests/属性测试.hl
测试 测试加法交换律() {
    // 生成随机输入
    对于 每个 a 在 随机.整数范围(-100, 100) 中
    对于 每个 b 在 随机.整数范围(-100, 100) 中
    {
        令 结果1 = a + b;
        令 结果2 = b + a;
        断言.等于(结果1, 结果2, "加法应该满足交换律");
    }
}

测试 测试乘法结合律() {
    对于 每个 a 在 随机.整数范围(1, 10) 中
    对于 每个 b 在 随机.整数范围(1, 10) 中
    对于 每个 c 在 随机.整数范围(1, 10) 中
    {
        令 结果1 = (a * b) * c;
        令 结果2 = a * (b * c);
        断言.等于(结果1, 结果2, "乘法应该满足结合律");
    }
}
```

## 基准测试

### 编写基准测试

```hl
// tests/基准测试.hl
基准 基准加法() {
    令 a = 1000;
    令 b = 2000;
    
    测量 {
        令 _ = a + b;
    }
}

基准 基准循环() {
    测量 {
        令 总和 = 0;
        令 i = 0;
        当 i < 1000 时
            总和 = 总和 + i;
            i = i + 1;
        结束
    }
}
```

## 调试技巧

### 常见错误类型

| 错误类型 | 描述 | 解决方法 |
|---------|------|---------|
| 语法错误 | 代码语法不正确 | 检查语法结构和关键词拼写 |
| 类型错误 | 类型不匹配 | 检查变量类型和函数参数 |
| 运行时错误 | 运行时发生异常 | 使用调试器和日志定位问题 |
| 逻辑错误 | 代码逻辑不正确 | 编写测试用例验证逻辑 |

### 调试工作流

1. **重现问题**：找到能够稳定重现问题的步骤
2. **定位问题**：使用调试器和日志缩小问题范围
3. **分析原因**：理解问题的根本原因
4. **验证修复**：编写测试用例验证修复效果
5. **回归测试**：确保修复不会引入新问题

### 性能调试

```hl
导入 "标准库/性能" 作为 性能;

函数 主函数() {
    // 性能分析
    性能.分析.开始();
    
    // 执行需要分析的代码
    令 结果 = 耗时操作();
    
    // 停止分析并生成报告
    性能.分析.停止();
    性能.分析.报告("分析报告.json");
}

函数 耗时操作() {
    令 总和 = 0;
    令 i = 0;
    当 i < 1000000 时
        总和 = 总和 + i;
        i = i + 1;
    结束
    返回 总和;
}
```

## 测试最佳实践

### 测试覆盖率

- **目标**：代码覆盖率应达到 80% 以上
- **重点**：测试核心功能和边界情况
- **工具**：使用 `huanlang test --coverage` 生成覆盖率报告

### 测试命名规范

- 测试函数名应清晰描述测试内容
- 使用 `测试_` 前缀标识测试函数
- 使用 `基准_` 前缀标识基准测试

### 测试组织

1. **按模块组织测试**：每个模块对应一个测试文件
2. **测试分类**：将测试分为单元测试、集成测试等
3. **测试数据**：使用有代表性的测试数据
4. **测试隔离**：确保测试之间相互独立

### 持续集成

```yaml
# .github/workflows/test.yml
name: Tests

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Run tests
      run: huanlang test --coverage
    - name: Upload coverage
      uses: codecov/codecov-action@v1
      with:
        file: ./coverage.xml
```

### 调试示例

```hl
// examples/debug_example.hl
导入 "标准库/控制台" 作为 控制台;
导入 "标准库/性能" 作为 性能;

函数 主函数() {
    性能.日志.启用();
    性能.日志.信息("程序开始");
    
    令 x = 10;
    性能.日志.调试("x 的初始值: " + x);
    
    令 y = 20;
    性能.日志.调试("y 的值: " + y);
    
    令 结果 = 计算(x, y);
    性能.日志.信息("计算结果: " + 结果);
    
    控制台.打印("最终结果: " + 结果);
    性能.日志.信息("程序结束");
}

函数 计算(a: 整数, b: 整数) -> 整数 {
    性能.日志.调试("进入计算函数, a: " + a + ", b: " + b);
    
    // 模拟复杂计算
    令 中间结果 = a * b;
    性能.日志.调试("中间结果: " + 中间结果);
    
    令 最终结果 = 中间结果 + 5;
    性能.日志.调试("最终结果: " + 最终结果);
    
    返回 最终结果;
}
```

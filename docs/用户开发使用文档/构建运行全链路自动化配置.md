## 1. 流程概述

幻语构建运行全链路自动化流程是一个集成了源代码识别、构建配置、编译执行和结果验证的完整解决方案。该流程通过 PowerShell 脚本实现，提供了从源代码到可执行程序运行的端到端自动化能力。

### 1.1 核心环节

1. **源代码文件识别与加载**：自动识别 `.hl` 和 `.幻` 后缀的源代码文件
2. **huan.exe 配置参数优化**：根据配置文件和命令行参数优化构建配置
3. **构建过程的错误捕获与处理**：实时捕获并处理构建过程中的错误
4. **编译结果的验证步骤**：验证编译是否成功完成
5. **可执行程序的运行环境配置**：设置运行环境变量和参数
6. **全链路可追溯性**：生成详细的日志文件记录关键操作

## 2. 工具与依赖

### 2.1 必要工具

- **PowerShell 5.1 或更高版本**：用于运行自动化脚本
- **huan.exe**：幻语编译器可执行文件
  - 发布版本：`target/release/huan.exe`
  - 调试版本：`target/debug/huan.exe`

### 2.2 脚本文件

- **huan_build_run.ps1**：主自动化脚本
- **huan_config.json**：默认配置文件
- **test_huan_build_run.ps1**：测试脚本

## 3. 配置选项

### 3.1 命令行参数

| 参数 | 类型 | 描述 | 默认值 |
|------|------|------|--------|
| `-SourcePath` | string | 源代码文件或目录路径 | 无（必填） |
| `-OutputPath` | string | 输出目录路径 | `./build_output` |
| `-ConfigFile` | string | 配置文件路径 | `./huan_config.json` |
| `-Release` | switch | 是否使用发布模式构建 | `false` |
| `-Verbose` | switch | 是否输出详细日志 | `false` |

### 3.2 配置文件结构

```json
{
  "build": {
    "optimize": "debug",      // 优化级别：debug 或 release
    "target": "native",       // 目标平台：native、wasm、llvm 等
    "features": []            // 启用的特性
  },
  "run": {
    "args": [],               // 运行参数
    "env": {                  // 环境变量
      "DEBUG": "false",
      "LOG_LEVEL": "info"
    }
  }
}
```

## 4. 执行流程

### 4.1 流程图

```

└─────────────────┘     └─────────────────┘     └─────────────────┘
          │                       │                       │
          ▼                       ▼                       ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 4. 检查 huan.exe │ <── │ 5. 构建项目     │ <── │ 6. 运行程序     │

          │                       │                       │
          ▼                       ▼                       ▼

│ 7. 生成日志     │ <── │ 8. 验证结果     │ <── │ 9. 清理环境     │
```

### 4.2 详细步骤

1. **初始化环境**
   - 创建输出目录
   - 初始化日志文件
   - 设置日志级别

2. **识别源代码**
   - 检查输入路径是否存在
   - 识别 `.hl` 和 `.幻` 文件
   - 验证文件数量

3. **加载配置**
   - 读取配置文件
   - 应用命令行参数覆盖
   - 验证配置有效性

4. **检查 huan.exe**
   - 检查发布版本是否存在（如果指定了 `-Release`）
   - 检查调试版本是否存在
   - 验证可执行权限

5. **构建项目**
   - 构建命令：`huan build <source> [options]`
   - 捕获构建输出
   - 检查构建结果

6. **运行程序**
   - 运行命令：`huan run <source> [args]`
   - 设置环境变量
   - 捕获运行输出

7. **生成日志**
   - 记录构建过程
   - 记录运行结果
   - 生成错误日志

8. **验证结果**
   - 检查构建是否成功
   - 检查运行是否成功
   - 验证输出文件

9. **清理环境**
   - 清理临时环境变量
   - 整理输出文件

10. **输出报告**
    - 显示执行结果
    - 提供日志路径

## 5. 错误处理机制

### 5.1 错误类型

| 错误类型 | 描述 | 处理方式 |
|---------|------|----------|
| 路径错误 | 源代码路径不存在 | 停止执行并提示错误 |
| 文件类型错误 | 不是幻语源代码文件 | 停止执行并提示错误 |
| 构建错误 | 构建过程失败 | 捕获错误信息并停止执行 |
| 运行错误 | 运行过程失败 | 捕获错误信息并停止执行 |
| 配置错误 | 配置文件无效 | 使用默认配置并警告 |

### 5.2 错误日志

错误信息会同时输出到：
- 控制台（红色字体）
- 主日志文件：`build_output/build_run.log`
- 错误日志文件：`build_output/error.log`

## 6. 日志系统

### 6.1 日志级别

- **Error**：错误信息（红色）
- **Warning**：警告信息（黄色）
- **Info**：信息（绿色）
- **Verbose**：详细信息（青色，仅 `-Verbose` 模式）

### 6.2 日志内容

每条日志包含：
- 时间戳：`yyyy-MM-dd HH:mm:ss.fff`
- 日志级别：`[Error]`, `[Warning]`, `[Info]`, `[Verbose]`
- 消息内容：详细的操作信息

### 6.3 日志文件

| 文件名 | 路径 | 描述 |
|-------|------|------|
| build_run.log | `output_dir/` | 主日志文件，包含所有操作信息 |
| error.log | `output_dir/` | 错误日志文件，仅包含错误信息 |
| run_output.txt | `output_dir/` | 程序运行输出 |

## 8. 最佳实践

### 8.1 配置建议

- **开发环境**：使用默认配置（debug 模式）
- **生产环境**：使用发布模式（`-Release` 参数）
- **大型项目**：使用项目目录方式，配合 `幻语包.toml` 配置

### 8.2 性能优化

- 对于频繁构建的项目，使用 `target/release/huan.exe` 获得更好性能
- 对于大型项目，考虑使用增量构建
- 合理设置 `features` 以减少构建时间

### 8.3 故障排查

当遇到问题时：
1. 检查 `error.log` 文件获取详细错误信息
2. 检查 `build_run.log` 文件了解完整执行过程
3. 启用 `-Verbose` 模式获取更详细的日志
4. 验证 huan.exe 是否正确构建

## 9. 示例用法

### 9.1 基本用法

```powershell
# 构建并运行单个文件
.uan_build_run.ps1 -SourcePath examples\basic\你好世界.hl

# 构建并运行项目目录
.uan_build_run.ps1 -SourcePath examples\basic_project

# 发布模式构建
.uan_build_run.ps1 -SourcePath examples\basic\你好世界.hl -Release

# 详细模式
.uan_build_run.ps1 -SourcePath examples\basic\你好世界.hl -Verbose

# 自定义输出目录
.uan_build_run.ps1 -SourcePath examples\basic\你好世界.hl -OutputPath my_output

# 自定义配置文件
.uan_build_run.ps1 -SourcePath examples\basic\你好世界.hl -ConfigFile my_config.json
```

### 9.2 高级配置示例

**自定义配置文件 `my_config.json`**：

```json
{
  "build": {
    "optimize": "release",
    "target": "native",
    "features": ["llvm-backend", "wasm-backend"]
  },
  "run": {
    "args": ["--debug", "--verbose"],
    "env": {
      "DEBUG": "true",
      "LOG_LEVEL": "debug",
      "API_KEY": "your_api_key"
    }
  }
}
```

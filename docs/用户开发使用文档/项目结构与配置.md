# 项目结构与配置

本文档介绍幻语项目的推荐结构和配置文件使用方法。

## 项目基本结构

一个标准的幻语项目通常包含以下结构：

```
项目名称/
├── src/
│   ├── lib.幻       # 库文件
│   ├── main.幻      # 主程序
│   └── 其他模块/    # 子模块
├── tests/
│   └── 测试文件.hl  # 测试文件
├── examples/
│   └── 示例文件.hl  # 示例文件
├── 幻语包.toml      # 项目配置文件
└── README.md        # 项目说明
```

## 配置文件

### 幻语包.toml

`幻语包.toml` 是幻语项目的配置文件，类似于 Rust 的 `Cargo.toml`。

#### 基本结构

```toml
[package]
name = "示例项目"
version = "0.1.0"
description = "一个幻语示例项目"
authors = ["开发者 <developer@example.com>"]
license = "MIT"

[dependencies]
# 依赖项
标准库 = "1.0"

[features]
# 特性标志
default = ["标准"]
标准 = []
高级 = []

[build]
# 构建配置
target = "native"
optimize = "release"
```

#### 字段说明

- **[package]**：项目基本信息
  - `name`：项目名称
  - `version`：版本号（遵循语义化版本）
  - `description`：项目描述
  - `authors`：作者信息
  - `license`：许可证

- **[dependencies]**：依赖项
  - 格式：`依赖名称 = "版本号"`
  - 支持 git 依赖：`依赖名称 = { git = "https://gitee.com/user/repo.git" }`

- **[features]**：特性标志
  - `default`：默认启用的特性
  - 其他特性可以通过 `--features` 标志启用

- **[build]**：构建配置
  - `target`：目标平台（native、wasm、llvm 等）
  - `optimize`：优化级别（debug、release）
  - `debug`：是否启用调试信息

### 工作区配置

对于多包项目，可以使用工作区配置：

```toml
[workspace]
members = [
    "core",
    "app",
    "utils"
]

[workspace.dependencies]
# 工作区共享依赖
标准库 = "1.0"
```

## 模块系统

### 模块定义

```hl
// src/lib.幻
模块 示例 {
    函数 问候() {
        打印("Hello from 示例模块");
    }
}

导出 示例;
```

### 模块导入

```hl
// src/main.幻
导入 "./lib" 作为 示例;

函数 主函数() {
    示例.问候();
}
```

### 目录结构与模块映射

```
src/
├── lib.幻        # 根模块
├──  utils/       # 子目录
│   ├── mod.幻    # 子模块入口
│   └── helper.幻 # 辅助函数
└── 网络/         # 网络模块
    ├── mod.幻
    └── client.幻
```

对应的导入方式：

```hl
导入 "./utils" 作为 工具;
导入 "./网络" 作为 网络;

工具.helper.函数();
网络.client.连接();
```

## 构建系统

### 命令行构建

```bash
# 构建项目
huanlang build

# 构建特定目标
huanlang build --target wasm

# 发布构建
huanlang build --release

# 清理构建产物
huanlang clean
```

### 构建配置选项

| 选项 | 描述 | 默认值 |
|------|------|--------|
| `--target` | 目标平台 | `native` |
| `--optimize` | 优化级别 | `debug` |
| `--features` | 启用的特性 | `default` |
| `--out-dir` | 输出目录 | `target` |
| `--manifest-path` | 配置文件路径 | `幻语包.toml` |

## 测试系统

### 测试文件结构

```
tests/
├── 单元测试.hl
├── 集成测试.hl
└── 端到端测试.hl
```

### 编写测试

```hl
// tests/单元测试.hl
测试 测试加法() {
    令 结果 = 1 + 1;
    断言.等于(结果, 2);
}

测试 测试减法() {
    令 结果 = 5 - 3;
    断言.等于(结果, 2);
}
```

### 运行测试

```bash
# 运行所有测试
huanlang test

# 运行特定测试
huanlang test 测试加法

# 运行测试并显示详细输出
huanlang test --verbose
```

## 包管理

### 安装依赖

```bash
# 添加依赖
huanlang add 依赖名称

# 添加特定版本
huanlang add 依赖名称@1.0.0

# 从 git 添加
huanlang add 依赖名称 --git https://gitee.com/user/repo.git
```

### 移除依赖

```bash
huanlang remove 依赖名称
```

### 更新依赖

```bash
# 更新所有依赖
huanlang update

# 更新特定依赖
huanlang update 依赖名称
```

### 依赖版本管理

| 版本格式 | 描述 | 示例 |
|---------|------|------|
| `^1.0.0` | 兼容更新 | 允许 1.x.x，不允许 2.0.0 |
| `~1.0.0` | 补丁更新 | 允许 1.0.x，不允许 1.1.0 |
| `1.0.0` | 精确版本 | 只允许 1.0.0 |
| `*` | 任意版本 | 允许任何版本 |

## 开发工具集成

### IDE 支持

幻语提供了以下 IDE 支持：

- **Visual Studio Code**：通过幻语语言服务器插件
- **IntelliJ IDEA**：通过幻语插件
- **内置终端编辑器**：`huanlang edit` 命令

### 语言服务器

```bash
# 启动语言服务器
huanlang lsp

# 检查 LSP 状态
huanlang lsp --status
```

### 代码格式化

```bash
# 格式化单个文件
huanlang format 文件名.hl

# 格式化整个项目
huanlang format .

# 检查格式（不修改）
huanlang format --check .
```

## 持续集成

### GitHub Actions 配置

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Build
      run: cargo build --release
    - name: Test
      run: cargo test
    - name: Format
      run: huanlang format --check .
```

### GitLab CI 配置

```yaml
# .gitlab-ci.yml
stages:
  - build
  - test
  - format

build:
  stage: build
  script:
    - cargo build --release

test:
  stage: test
  script:
    - cargo test

format:
  stage: format
  script:
    - huanlang format --check .
```

## 最佳实践

### 项目组织

1. **模块化设计**：将代码按照功能划分为多个模块
2. **清晰的目录结构**：使用有意义的目录名称
3. **统一的命名规范**：遵循幻语的命名约定
4. **详细的文档**：为公共 API 编写文档

### 配置管理

1. **环境变量**：使用环境变量管理敏感配置
2. **配置文件**：使用 `幻语包.toml` 管理项目配置
3. **多环境配置**：为不同环境（开发、测试、生产）创建不同的配置

### 依赖管理

1. **版本锁定**：使用 `幻语包.lock` 锁定依赖版本
2. **依赖更新**：定期更新依赖以获取安全修复
3. **最小依赖**：只添加必要的依赖

## 示例项目结构

### 完整的项目示例

```
my-project/
├── src/
│   ├── main.幻           # 主程序
│   ├── lib.幻            # 库文件
│   ├── 模型/            # 数据模型
│   │   ├── mod.幻       # 模型模块
│   │   └── user.幻      # 用户模型
│   ├── 服务/            # 业务逻辑
│   │   ├── mod.幻       # 服务模块
│   │   └── auth.幻      # 认证服务
│   └── 工具/            # 工具函数
│       ├── mod.幻       # 工具模块
│       └── helper.幻    # 辅助函数
├── tests/
│   ├── 模型测试.hl      # 模型测试
│   ├── 服务测试.hl      # 服务测试
│   └── 集成测试.hl      # 集成测试
├── examples/
│   └── 使用示例.hl      # 使用示例
├── 幻语包.toml          # 项目配置
├── 幻语包.lock          # 依赖锁定
└── README.md            # 项目说明
```

通过遵循这些最佳实践，您可以创建结构清晰、易于维护的幻语项目。

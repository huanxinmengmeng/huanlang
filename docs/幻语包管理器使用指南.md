# 幻语包管理器使用指南

## 1. 概述

幻语包管理器（HLPM）是一个功能强大的包管理工具，用于管理幻语项目的依赖、构建和发布。它支持以下核心功能：

- 项目初始化和管理
- 依赖解析和版本控制
- 包的发布和安装
- 工作区管理
- 功能标志支持
- 外部库集成

## 2. 安装

### 2.1 从源码构建

```bash
# 克隆仓库
git clone https://gitee.com/huanxinmengmeng/huanlang.git
cd huanlang

# 构建包管理器
cargo build --release

# 安装到系统路径
sudo cp target/release/hlpm /usr/local/bin/
```

### 2.2 从预编译二进制文件安装

下载对应平台的二进制文件，然后添加到系统路径即可。

## 3. 基本命令

### 3.1 项目初始化

```bash
# 初始化新项目
hlpm init my-project

# 指定版本
hlpm init my-project --edition 1.2
```

### 3.2 依赖管理

```bash
# 添加依赖
hlpm add 网络@0.3 序列化@1.0

# 添加开发依赖
hlpm add 测试工具@0.5 --dev

# 移除依赖
hlpm remove 网络

# 更新依赖
hlpm update

# 安装依赖
hlpm install
```

### 3.3 构建和测试

```bash
# 构建项目
hlpm build

# 运行测试
hlpm test

# 运行项目
hlpm run
```

### 3.4 发布和搜索

```bash
# 发布项目
hlpm publish

# 搜索包
hlpm search 网络

# 查看包信息
hlpm info 网络
```

### 3.5 工作区命令

```bash
# 添加工作区成员
hlpm workspace add member1

# 移除工作区成员
hlpm workspace remove member1

# 列出工作区成员
hlpm workspace list
```

### 3.6 其他命令

```bash
# 清理缓存
hlpm clean

# 查看帮助
hlpm help

# 查看版本
hlpm version
```

## 4. 包描述文件格式

### 4.1 基本结构

```toml
[package]
name = "my-package"
version = "0.1.0"
edition = "1.2"
authors = ["Your Name <you@example.com>"]
description = "A description of your package"
license = "MIT"

[dependencies]
网络 = "0.3"
序列化 = { version = "1.0", optional = true }

[dev-dependencies]
测试工具 = "0.5"

[build-dependencies]
构建工具 = "0.2"

[features]
default = ["std"]
std = []
json = ["序列化"]
```

### 4.2 高级配置

```toml
[lib]
name = "my_package"
path = "src/lib.幻"

[[bin]]
name = "my-app"
path = "src/main.幻"

[examples]
hello = {
    name = "hello",
    path = "examples/hello.幻"
}

[tests]
test_basic = {
    name = "test_basic",
    path = "tests/test_basic.幻"
}

[benches]
benchmark = {
    name = "benchmark",
    path = "benches/benchmark.幻"
}

[build]
opt_level = 2
target = "x86_64-pc-windows-msvc"

[profile.dev]
opt_level = 0
debug = true

[profile.release]
opt_level = 3
debug = false
lto = true

[workspace]
members = ["core", "utils", "app"]

[workspace.dependencies]
网络 = "0.3"
序列化 = "1.0"
```

## 5. 工作区管理

### 5.1 创建工作区

在项目根目录创建 `幻语包.toml` 文件，添加 `[workspace]` 部分：

```toml
[package]
name = "workspace"
version = "0.1.0"

[workspace]
members = ["core", "utils", "app"]

[workspace.dependencies]
网络 = "0.3"
序列化 = "1.0"
```

### 5.2 工作区成员配置

每个成员包可以使用工作区依赖：

```toml
[package]
name = "core"
version = "0.1.0"

[dependencies]
网络 = { workspace = true }
```

## 6. 功能标志

### 6.1 定义功能

```toml
[features]
default = ["std"]
std = []
network = ["网络"]
json = ["序列化"]
db = ["数据库"]
full = ["network", "json", "db"]
```

### 6.2 使用功能

在代码中使用条件导入：

```rust
如果 特性("network") {
    导入 网络
}

导出 函数 发送请求(网址: 字符串) -> 可选<字符串> {
    如果 特性("network") {
        // 实际的网络请求
        返回 一些("成功")
    } 否则 {
        返回 无
    }
}
```

### 6.3 启用功能

```bash
# 启用特定功能
hlpm build --features network,json

# 启用所有功能
hlpm build --features full
```

## 7. 依赖解析

### 7.1 版本约束

- `^1.2.3` - 兼容版本（1.x.x）
- `~1.2.3` - 补丁版本（1.2.x）
- `*` - 任意版本
- `1.2.*` - 1.2.x 版本
- `>=1.2.3` - 大于等于 1.2.3
- `>1.2.3` - 大于 1.2.3
- `<=1.2.3` - 小于等于 1.2.3
- `<1.2.3` - 小于 1.2.3
- `1.2.3..1.3.0` - 版本范围
- `1.2.3 || 1.3.0` - 多个版本

### 7.2 依赖来源

- 注册表：`网络 = "0.3"`
- 本地路径：`网络 = { path = "../网络" }`
- Git 仓库：`网络 = { git = "https://gitee.com/huanxinmengmeng/网络.git", branch = "main" }`

## 8. 缓存管理

### 8.1 缓存目录

- Windows: `%USERPROFILE%\.huanlang\cache`
- Linux/Mac: `~/.huanlang/cache`

### 8.2 清理缓存

```bash
# 清理30天前的缓存
hlpm clean

# 清理7天前的缓存
hlpm clean --days 7
```

## 10. 常见问题

### 10.1 依赖冲突

当遇到依赖冲突时，包管理器会显示详细的冲突信息，你可以：

- 手动指定版本：`网络 = "1.0.0"`
- 使用解析器覆盖：`resolver = "2"`
- 排除冲突依赖：`网络 = { version = "1.0", default-features = false }`

### 10.2 构建失败

- 检查依赖是否正确安装
- 检查构建脚本是否有错误
- 检查系统环境是否满足要求

### 10.3 网络问题

- 检查网络连接
- 配置代理：`export HTTP_PROXY=http://proxy.example.com:8080`
- 使用本地缓存：`hlpm install --offline`

## 11. 最佳实践

### 11.1 项目结构

```
my-project/
├── 幻语包.toml       # 包描述文件
├── 幻语锁.toml       # 依赖锁定文件
├── src/
│   ├── lib.幻        # 库代码
│   └── main.幻       # 可执行文件
├── examples/         # 示例代码
├── tests/            # 测试代码
└── benches/          # 基准测试
```

### 11.2 版本管理

- 使用语义化版本：`主版本.次版本.补丁版本`
- 主版本：不兼容的API变更
- 次版本：向后兼容的功能添加
- 补丁版本：向后兼容的错误修复

### 11.3 依赖管理

- 明确指定版本范围
- 使用锁定文件确保构建一致性
- 定期更新依赖以获取安全补丁
- 避免过度依赖外部包


## 13. 命令参考

### 13.1 全局选项

- `--help` - 显示帮助信息
- `--version` - 显示版本信息
- `--verbose` - 显示详细输出
- `--quiet` - 仅显示错误信息
- `--color` - 启用或禁用颜色

### 13.2 命令列表

| 命令 | 描述 | 选项 |
|------|------|------|
| `init` | 初始化新项目 | `--edition` - 指定幻语版本 |
| `add` | 添加依赖 | `--dev` - 添加为开发依赖<br>`--build` - 添加为构建依赖 |
| `remove` | 移除依赖 | - |
| `update` | 更新依赖 | `--all` - 更新所有依赖 |
| `install` | 安装依赖 | `--offline` - 离线模式 |
| `build` | 构建项目 | `--features` - 启用功能<br>`--release` - 发布模式 |
| `test` | 运行测试 | `--features` - 启用功能 |
| `run` | 运行项目 | `--features` - 启用功能<br>`--args` - 传递参数 |
| `publish` | 发布项目 | `--version` - 指定版本 |
| `search` | 搜索包 | `--limit` - 限制结果数量 |
| `info` | 显示信息 | `--versions` - 显示版本列表 |
| `clean` | 清理缓存 | `--days` - 清理天数 |
| `workspace` | 工作区命令 | `add` - 添加成员<br>`remove` - 移除成员<br>`list` - 列出成员 |

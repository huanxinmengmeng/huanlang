# 幻语编程语言标准规范

**版本 1.0（草案）**  
**基于幻语 v0.0.1 设计**  
**最后更新：2026年4月**

---

## 前言

幻语（HuanLang）是一门面向全场景的现代系统编程语言，致力于提供**中文原生**的开发体验，同时保持**零成本抽象**、**无垃圾回收**和**底层控制力**。语言设计注重**人类可读性**、**机器执行效率**和**AI 协作友好性**，追求覆盖从微控制器到云服务器的全谱系应用。

本规范定义了幻语 1.0 的语法、语义、标准库和行为约束，是所有符合标准实现（编译器、解释器、工具链）的根本依据。

---

## 1. 语言基础

### 1.1 文件格式与编码
- 源文件扩展名：`.hl`
- AI 交换格式扩展名：`.hla`
- 模块接口文件：`.hlm`
- 独立汇编文件：`.hasm`
- 所有文件必须使用 UTF-8 无 BOM 编码。
- 换行符：LF（`\n`）推荐，兼容 CRLF。
- 标识符允许 Unicode 字符（CJK 统一表意字符、字母、数字、下划线），数字不可作为首字符。

### 1.2 词法元素
#### 1.2.1 三语关键词
幻语同时支持**中文**、**拼音**和**英文**三种关键词体系，内部统一映射到同一语义，可以自由混用。以下列出部分核心关键词对标表（完整列表见附录A）。

| 中文 | 拼音 | 英文 | 语义 |
|------|------|------|------|
| 令 | ling | let | 变量声明 |
| 定 | ding | const | 常量声明 |
| 为 | wei | be | 声明赋值 |
| 若 | ruo | if | 条件分支 |
| 则 | ze | then | 条件 then |
| 否则 | fouze | else | 条件 else |
| 结束 | jieshu | end | 块结束 |
| 函数 | hanshu | func | 函数定义 |
| 返回 | fanhui | return | 返回 |
| 结构 | jiegou | struct | 结构体 |
| 字段 | ziduan | field | 字段 |
| 模块 | mokuai | module | 模块 |
| 导入 | daoru | import | 导入 |
| 真 | zhen | true | 布尔真 |
| 假 | jia | false | 布尔假 |
| 空 | kong | null | 空值 |
| 加 | jia | add | 加法运算符 |
| 大于 | dayu | gt | 大于运算符 |

类型关键词同样支持三语（如 `整数`/`zhengshu`/`int` 等）。

#### 1.2.2 字面量
- 整数：十进制 `123`、`1_000`；十六进制 `0x1A`；二进制 `0b1010`；八进制 `0o77`
- 浮点数：`3.14`、`2.99e8`
- 字符：`'A'`、`'中'`，支持转义 `\n`、`\u{4E2D}`
- 字符串：`"Hello, 世界"`，支持转义
- 布尔：`真` 或 `true`、`假` 或 `false`
- 空：`空` 或 `null`

#### 1.2.3 注释
- 单行：`-- 注释内容`
- 多行：`/* 注释 */`，可嵌套

### 1.3 程序结构
幻语程序由一系列**顶层项**组成，包括函数、结构体、特征（trait）、实现、模块、导入、外部块、类型别名和全局变量。

```
程序 ::= { 顶层项 }
顶层项 ::= 导入语句 | 模块 | 函数 | 结构体 | 特征 | 实现 | 外部块 | 类型别名 | 全局变量
```

模块由 `模块 <名称> { ... }` 定义，支持嵌套。可见性默认为私有，使用 `公开`（pub）导出。

## 2. 数据类型与类型系统

### 2.1 基本类型
| 类型 | 说明 | 大小（字节） |
|------|------|------------|
| `整数`（int） | 平台相关有符号整数，通常 64 位 | 8 |
| `整数8/16/32/64` | 定长有符号整数 | 1,2,4,8 |
| `无符号8/16/32/64` | 定长无符号整数 | 1,2,4,8 |
| `浮点32/64` | IEEE 浮点数 | 4,8 |
| `布尔` | 真/假 | 1 |
| `字符` | Unicode 标量值 | 4 |
| `字符串` | UTF-8 字符串，不可变 | 24 |
| `单元` | 空类型，类似 `()` | 0 |

### 2.2 复合类型
- `列表<元素类型>`：动态数组
- `数组<元素类型; 长度>`：固定长度数组
- `字典<键类型, 值类型>`：哈希表
- `指针<类型>`：原始指针
- `可选<类型>`：可选值，用于表达可能为空的值
- `函数(参数类型列表) -> 返回类型`：函数类型

### 2.3 自定义类型
- **结构体**：`结构 名称 { 字段 名称 : 类型 ... }`，支持方法定义
- **特征**：定义共享行为接口，类似 Rust trait，可包含默认实现
- **枚举**：代数数据类型，带有数据的变体
- **类型别名**：`类型 新名称 = 已有类型`

### 2.4 泛型
函数、结构体和特征支持泛型参数。编译器通过单态化（monomorphization）生成具体代码，无运行时开销。

### 2.5 类型推导
基于 Hindley-Milner 算法，支持 let 多态。大多数局部变量无需显式指定类型。

## 3. 表达式与运算符

### 3.1 运算符列表
按优先级从高到低：

| 优先级 | 运算符 | 结合性 |
|--------|--------|--------|
| 1 | `非`、`-`（负号）、`按位非` | 右结合 |
| 2 | `乘`、`除`、`取余` | 左结合 |
| 3 | `加`、`减` | 左结合 |
| 4 | `左移`、`右移` | 左结合 |
| 5 | `按位与` | 左结合 |
| 6 | `按位异或` | 左结合 |
| 7 | `按位或` | 左结合 |
| 8 | `大于`、`小于`、`等于`、`不小于`、`不大于`、`不等于` | 左结合 |
| 9 | `且` | 左结合 |
| 10 | `或` | 左结合 |

赋值运算符 `设为`（`assign`）优先级最低。

### 3.2 字面量构造
- 列表：`[1, 2, 3]`
- 字典：`{"键": 值, ...}`
- 结构体：`结构名 { 字段设为 值, ... }`
- 闭包：`|参数| 表达式`

### 3.3 控制表达式
- 条件表达式：`若 条件 则 表达式1 否则 表达式2`
- 模式匹配表达式：`匹配 值 { 模式 => 表达式, ... }`
- 可选链：`表达式?`（对可选类型的短路操作）

## 4. 语句

### 4.1 变量与常量声明
```hl
令 变量名 [类型 类型] 为 表达式
定 常量名 [类型 类型] 为 表达式
```

### 4.2 控制流语句
- `若 条件 则 { ... } [否则若 条件 ... ] [否则 { ... }] 结束`
- `匹配 表达式 { 当 模式 => { ... } ... [默认 => { ... }] }`
- `重复 次数 次 { ... } 结束`
- `当 条件 循环 { ... } 结束`
- `对于 每个 变量 在 容器 中 { ... } 结束`
- `跳出` 和 `继续` 用于循环控制

### 4.3 函数定义与调用
```hl
[公开] 函数 函数名<泛型>(参数: 类型, ...) [返回 类型]
    [前置: 条件]
    [后置: 条件]
开始
    语句...
结束
```
函数可嵌套定义内部函数。

### 4.4 错误处理
- 错误值使用 `结果<成功类型, 错误类型>` 或 `可选<类型>`。
- `?` 操作符：若表达式为错误或空，则提前返回；否则解包成功值。

## 5. 内存管理

### 5.1 分配器模型
- 无垃圾回收，所有堆分配必须通过**分配器**显式进行。
- 标准分配器特征：
  ```
  特征 分配器
      需要 分配(大小: 整数, 对齐: 整数) -> 指针<单元>
      需要 释放(指针: 指针<单元>, 大小: 整数, 对齐: 整数)
  ```
- 全局默认分配器：`幻语.内存.默认分配器()`

### 5.2 所有权模式（可选）
通过 `使用 所有权` 或编译器标志 `--ownership` 启用。所有权规则：
- 每个值只一个所有者。
- 赋值、传参默认移动所有权。
- 基本类型自动实现 `复制` 特征，赋值时复制。
- 借用分为只读借用（`&T`）和可变借用（`&mut T`），借用规则确保无数据竞争。

### 5.3 智能指针
标准库提供 `盒子<T>`（堆分配）、`引用计数<T>`（共享所有权）、`弱引用<T>`。

## 6. 并发模型

### 6.1 结构化并发
任务通过**任务组**组织，形成父子树。父任务生命周期覆盖所有子任务，子任务失败自动取消兄弟任务并传播错误。

### 6.2 通道
基于 CSP 模型的通道，支持有界/无界缓冲，多生产者多消费者。

### 6.3 同步原语
提供互斥锁、读写锁、原子变量、屏障、一次性执行等。

### 6.4 线程安全
通过 `发送`（Send）和 `同步`（Sync）特征标记，编译器自动推导，保障编译期无数据竞争。

---

## 7. 汇编与裸机编程

### 7.1 内联汇编
使用 `汇编!(...)` 语法，支持模板字符串、操作数约束、破坏列表和选项。支持 x86、ARM、RISC-V 等架构。

### 7.2 独立汇编文件
以 `.hasm` 为扩展名，采用 GNU 汇编语法，可定义中断向量表、启动代码等。

### 7.3 外设寄存器
使用 `外设 <名称> 基址 <地址>:` 语法定义寄存器块，支持位域操作。

### 7.4 链接器脚本
提供声明式的链接脚本语言，可定义内存布局、段分配等，最终生成平台对应的 LD 脚本。

### 7.5 中断服务
`@中断` 和 `@调用约定("中断")` 属性用于标记 ISR，编译器自动生成上下文保存/恢复代码。

## 8. 跨语言互操作

### 8.1 外部函数接口（FFI）
通过 `外部 "<语言>"` 块调用 C、Python、Java、JavaScript 等语言的库。支持动态绑定、静态绑定和完全转译三种集成级别。

### 8.2 多语言绑定生成
基于 UniFFI 理念，从幻语接口文件自动生成 Kotlin、Swift、Python、Go 等语言的原生绑定。

### 8.3 代码转译器
提供 `幻语 ↔ 目标语言` 的双向转译框架，支持 C、C++、Rust、Python、Java、Go、JavaScript、TypeScript 等十余种语言。

---

## 9. 编译器架构与中间表示

幻语编译器采用 **双前端 + MLIR + 多后端** 架构：
1. **人类前端**（`.hl` 解析）和 **AI 前端**（`.hla` 解析）分别生成统一 AST。
2. AST 经语义分析后，转换为 **幻语 MLIR 方言**。
3. 方言经过一系列降级 Pass 转换为标准 MLIR 方言（scf、arith、func，最终至 llvm）。
4. **LLVM 后端**生成原生机器码；**Wasm 后端**生成 WebAssembly；**GPU 后端**生成 CUDA/HIP 内核。

MLIR 方言的定义和降级规则在幻语编译器源码的 `dialects/` 目录中。

## 10. 包管理与工具链

### 10.1 包管理器
`huan package` 命令管理依赖、构建、发布。包描述文件为 `幻语包.toml`，支持语义版本、依赖解析、工作区等功能。

### 10.2 命令行接口
- `huan build`：编译
- `huan run`：编译并运行
- `huan test`：运行测试
- `huan fmt`：格式化与关键词风格转换
- `huan edit`：启动内置终端编辑器
- `huan serve`：启动 LSP 服务器
- `huan transpile`：跨语言转换
- `huan import-lib`：导入外部库

### 10.3 内置编辑器
一个基于终端的文本编辑器（类 Vim），支持语法高亮、LSP 集成、多光标等。

### 10.4 LSP 服务器
实现标准 LSP 协议，并扩展 AI 代码生成、代码解释、AEF 转换等自定义方法。

## 11. AI 交换格式（`.hla`）

### 11.1 目的
为 AI 模型提供一种**无歧义、行独立、紧凑**的幻语代码表示，便于生成和解析。

### 11.2 格式语法
每行一个操作，可选标签。
```
[标签] 操作码 操作数...
```
操作码采用拼音助记符，如 `LING`（变量声明）、`HANSHU`（函数）、`DIAOYONG`（调用）。

### 11.3 类型与值表示
类型和值以字符串形式内联，如 `整数`、`"hello"`、`真`。

### 11.4 双向转换
幻语源码（`.hl`）与 `.hla` 可通过编译器无损互转。

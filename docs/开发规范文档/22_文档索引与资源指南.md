## 22.4 问题与贡献

### 报告问题

若发现编译器 bug、文档错误或有功能建议，**请在 Gitee 主仓库的 Issues 页面提交**：

- Gitee Issues: https://gitee.com/huanxinmengmeng/huanlang/issues

提交问题时请提供以下信息：
- 幻语编译器版本（`huan version`）
- 操作系统及版本
- 可复现问题的最小代码示例
- 期望行为与实际行为的描述

### 贡献代码

欢迎开发者贡献代码！请遵循以下流程：

1. **Fork Gitee 仓库**：在 Gitee 上 fork 项目（推荐从主仓库操作）。
2. **创建分支**：从 `main` 分支创建功能分支，命名格式：`feature/功能描述` 或 `fix/问题描述`。
3. **编写代码**：
   - Rust 代码遵循 `rustfmt` 格式化
   - 幻语代码使用 `huan fmt` 统一风格
   - 新功能必须包含测试用例
4. **提交变更**：提交信息格式：`[模块] 简短描述`，例如 `[编译器] 修复类型推导循环引用问题`
5. **发起 Pull Request**：在 Gitee 主仓库发起 PR。

## 22.7 常见问题（FAQ）

**Q1: 幻语适合哪些应用场景？**

A: 幻语是一门全栈语言，适用于：
- 系统编程（操作系统内核、驱动）
- 嵌入式开发（单片机、裸机）
- 后端服务（Web服务器、微服务）
- 桌面应用（跨平台GUI）
- AI应用开发（模型推理、Agent）

**Q2: 幻语与 Rust/C++ 相比如何？**

A: 性能方面，幻语通过LLVM生成与Rust/C++同级的机器码。语法方面，幻语提供中文原生支持，更适合中文开发者。内存安全方面，所有权模式提供了类似Rust的安全保障，但默认模式更灵活。

**Q3: 如何将现有C/C++项目迁移到幻语？**

A: 可通过以下步骤逐步迁移：
1. 使用 `huan import-lib` 导入C/C++库生成绑定
2. 用幻语重写部分模块，通过FFI与原代码交互
3. 使用 `huan transpile` 辅助转换简单函数
4. 逐步替换，直到完全迁移

**Q4: 幻语的学习曲线如何？**

A: 对于有编程经验的开发者，掌握基础语法约需1-2天。中文关键词降低了记忆负担。所有权系统对于熟悉Rust的开发者是熟悉的，新开发者可在默认模式下先忽略，逐步学习。

**Q5: 目前有没有在线Playground可以尝试？**

A: 暂时没有独立的在线Playground。您可以在本地克隆仓库并构建编译器，或下载发布版本的二进制文件进行尝试。我们也欢迎社区贡献者帮助搭建在线Playground。

**Q6: 为什么有两个仓库？我应该用哪个？**

A: Gitee 是国内访问更快的主仓库，所有开发活动和 Issue 跟踪均在此进行。GitHub 是自动同步的镜像，方便国际用户访问和参与。建议国内开发者优先使用 Gitee。

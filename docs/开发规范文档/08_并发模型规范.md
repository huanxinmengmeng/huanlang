# 8. 并发模型规范

## 8.1 并发模型概述

幻语采用**结构化并发（Structured Concurrency）**作为核心并发模型，并辅以 **CSP（Communicating Sequential Processes）** 风格的通道通信。所有并发原语均与所有权系统无缝集成，在编译期保障数据竞争自由。

**核心设计原则**：
- **任务树结构**：并发任务形成父子层级，父任务的生命周期覆盖所有子任务。
- **错误自动传播**：子任务失败时自动取消兄弟任务，并将错误向上传播。
- **无数据竞争**：通过 `发送`（Send）和 `同步`（Sync）特征在编译期保证线程安全。
- **零成本抽象**：高级并发 API 在编译后展开为高效的系统调用或线程池操作，无额外运行时开销。

## 8.3 结构化并发 API

### 8.3.1 任务与任务组

任务（Task）是幻语并发的执行单元。任务组（TaskGroup）管理一组子任务，确保在任务组离开作用域前所有子任务已完成。

**核心类型定义**：
```rust
/// 任务标识符
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct TaskId(u64);

/// 任务句柄，用于获取任务结果或取消任务
pub struct TaskHandle<T> {
    id: TaskId,
    receiver: oneshot::Receiver<Result<T, TaskError>>,
}

/// 任务组，管理子任务的生命周期
pub struct TaskGroup {
    handles: Vec<JoinHandle<()>>,
    cancel_sender: Option<oneshot::Sender<()>>,
}
```

### 8.3.2 任务组实现接口

```rust
impl TaskGroup {
    /// 创建新的任务组
    pub fn new() -> Self {
        Self {
            handles: Vec::new(),
            cancel_sender: None,
        }
    }

    /// 在任务组内创建子任务
    pub fn spawn<F, T>(&mut self, f: F) -> TaskHandle<T>
    where
        F: FnOnce() -> T + Send + 'static,
        T: Send + 'static,
    {
        let (tx, rx) = oneshot::channel();
        let handle = std::thread::spawn(move || {
            let result = f();
            let _ = tx.send(Ok(result));
        });
        self.handles.push(handle);
        TaskHandle {
            id: TaskId(0), // 实际需分配唯一 ID
            receiver: rx,
        }
    }

    /// 等待所有子任务完成
    pub fn wait_all(self) -> Vec<Result<(), JoinError>> {
        self.handles
            .into_iter()
            .map(|h| h.join())
            .collect()
    }
}

impl<T> TaskHandle<T> {
    /// 获取任务结果（阻塞直到任务完成）
    pub fn result(self) -> Result<T, TaskError> {
        self.receiver.recv().unwrap_or(Err(TaskError::Cancelled))
    }
}

/// 在任务组离开作用域时自动等待
impl Drop for TaskGroup {
    fn drop(&mut self) {
        for handle in self.handles.drain(..) {
            let _ = handle.join();
        }
    }
}
```

## 8.4 通道（CSP 模型）

通道提供任务间的消息传递，支持有界缓冲和无界缓冲。

### 8.4.1 通道 API

```rust
pub struct Channel<T> {
    sender: Sender<T>,
    receiver: Receiver<T>,
}

impl<T> Channel<T> {
    /// 创建有界通道
    pub fn bounded(capacity: usize) -> Self {
        let (tx, rx) = mpsc::sync_channel(capacity);
        Self {
            sender: Sender(tx),
            receiver: Receiver(rx),
        }
    }

    /// 创建无界通道
    pub fn unbounded() -> Self {
        let (tx, rx) = mpsc::channel();
        Self {
            sender: Sender(tx),
            receiver: Receiver(rx),
        }
    }

    /// 获取发送端
    pub fn sender(&self) -> Sender<T> {
        self.sender.clone()
    }

    /// 获取接收端
    pub fn receiver(&self) -> Receiver<T> {
        self.receiver.clone()
    }
}

pub struct Sender<T>(mpsc::SyncSender<T>);

impl<T> Sender<T> {
    /// 发送消息（阻塞）
    pub fn send(&self, value: T) -> Result<(), SendError<T>> {
        self.0.send(value)
    }

    /// 尝试发送（非阻塞）
    pub fn try_send(&self, value: T) -> Result<(), TrySendError<T>> {
        self.0.try_send(value)
    }
}

pub struct Receiver<T>(mpsc::Receiver<T>);

impl<T> Receiver<T> {
    /// 接收消息（阻塞）
    pub fn recv(&self) -> Result<T, RecvError> {
        self.0.recv()
    }

    /// 尝试接收（非阻塞）
    pub fn try_recv(&self) -> Result<T, TryRecvError> {
        self.0.try_recv()
    }

    /// 迭代接收消息（阻塞，直到通道关闭）
    pub fn iter(&self) -> Iter<T> {
        Iter { receiver: self }
    }
}

impl<T> Iterator for Iter<'_, T> {
    type Item = T;
    fn next(&mut self) -> Option<Self::Item> {
        self.receiver.recv().ok()
    }
}
```

### 8.5.1 互斥锁（Mutex）

```rust
pub struct Mutex<T> {
    inner: std::sync::Mutex<T>,
}

impl<T> Mutex<T> {
    pub fn new(value: T) -> Self {
        Self {
            inner: std::sync::Mutex::new(value),
        }
    }

    /// 获取锁，返回守卫
    pub fn lock(&self) -> MutexGuard<T> {
        MutexGuard {
            guard: self.inner.lock().unwrap(),
        }
    }

    /// 尝试获取锁
    pub fn try_lock(&self) -> Option<MutexGuard<T>> {
        self.inner.try_lock().ok().map(|g| MutexGuard { guard: g })
    }
}

pub struct MutexGuard<'a, T> {
    guard: std::sync::MutexGuard<'a, T>,
}

impl<T> Deref for MutexGuard<'_, T> {
    type Target = T;
    fn deref(&self) -> &Self::Target {
        &self.guard
    }
}

impl<T> DerefMut for MutexGuard<'_, T> {
    fn deref_mut(&mut self) -> &mut Self::Target {
        &mut self.guard
    }
}
```

## 8.6 数据竞争保护：Send 与 Sync 特征

幻语的所有权系统与并发模型深度集成，通过 `发送`（Send）和 `同步`（Sync）两个自动特征在编译期保障线程安全。

### 8.6.1 特征定义

```rust
/// 标记类型可以安全地跨任务转移所有权
/// 
/// # 安全性
/// 实现此特征的类型必须保证其值可以安全地从一个线程移动到另一个线程。
pub unsafe auto trait Send {}

/// 标记类型可以安全地跨任务共享引用
/// 
/// # 安全性
/// 实现此特征的类型必须保证其值可以安全地被多个线程同时访问（只读共享）。
pub unsafe auto trait Sync {}
```

### 8.6.2 自动实现规则

编译器根据以下规则自动为类型实现 `Send` 和 `Sync`：

- 所有基本数值类型（`整数`、`浮点32` 等）自动实现 `Send` 和 `Sync`
- 若类型的所有字段均为 `Send`，则该类型自动实现 `Send`
- 若类型的所有字段均为 `Sync`，则该类型自动实现 `Sync`
- 原始指针（`指针<T>`）不实现 `Send` 和 `Sync`
- `互斥锁<T>` 若 `T` 为 `Send` 则实现 `Send` 和 `Sync`
- `原子` 类型均实现 `Send` 和 `Sync`

### 8.6.3 在 API 中使用 Send 约束

任务创建函数要求闭包类型必须为 `Send`：

```rust
impl TaskGroup {
    pub fn spawn<F, T>(&mut self, f: F) -> TaskHandle<T>
    where
        F: FnOnce() -> T + Send + 'static,  // 闭包必须可跨线程发送
        T: Send + 'static,                   // 返回值必须可跨线程发送
    {
        // ...
    }
}
```

通道的 `Sender<T>` 要求 `T` 为 `Send`：

```rust
impl<T: Send> Sender<T> {
    pub fn send(&self, value: T) -> Result<(), SendError<T>> {
        self.0.send(value)
    }
}
```

## 8.7 运行时调度

幻语支持两种任务调度模型，可通过编译选项切换：

| 模型 | 标志 | 描述 |
|------|------|------|
| **1:1 线程模型** | 默认 | 每个幻语任务对应一个操作系统线程 |
| **M:N 异步模型** | `--运行时 异步` | 使用 tokio 或自定义运行时，多个任务在少量线程上调度 |

### 8.7.1 异步运行时集成（可选）

当使用 `--运行时 异步` 时，任务组创建的是异步任务：

```rust
#[cfg(feature = "async")]
pub struct AsyncTaskGroup {
    handles: Vec<tokio::task::JoinHandle<()>>,
}

#[cfg(feature = "async")]
impl AsyncTaskGroup {
    pub fn new() -> Self {
        Self { handles: Vec::new() }
    }

    pub fn spawn<F, T>(&mut self, f: F) -> AsyncTaskHandle<T>
    where
        F: std::future::Future<Output = T> + Send + 'static,
        T: Send + 'static,
    {
        let (tx, rx) = oneshot::channel();
        let handle = tokio::spawn(async move {
            let result = f.await;
            let _ = tx.send(Ok(result));
        });
        self.handles.push(handle);
        AsyncTaskHandle { receiver: rx }
    }

    pub async fn wait_all(self) {
        for handle in self.handles {
            let _ = handle.await;
        }
    }
}
```


# 幻语文档统一风格指南

## 1. 文档结构规范

### 1.1 标题层级
- 一级标题 (`#`)：章节标题（如 `# 1. 语言标识与文件格式`）
- 二级标题 (`##`)：主要节（如 `## 1.1 语言标识符`）
- 三级标题 (`###`)：子节（如 `### 1.2.1 .hl 源代码文件`）
- 四级标题 (`####`)：细项说明（如 `#### Shebang 处理规范`）

### 1.2 统一头部
所有章节文档应包含以下头部：
```markdown
# 幻语编程语言 - 开发规范文档
# 章节标题

**版本**: v0.4.0  
**许可证**: Apache-2.0  
**更新日期**: 2026年5月  
**适用对象**: 编译器开发者、工具链开发者

> **说明**: 本章详细描述 [主题内容]，提供精确的实现指导。
```

### 1.3 分隔线
- 使用 `---` 作为主要节之间的分隔线
- 分隔线前后各留一空行

### 1.4 文档结尾
- 所有章节文档结尾添加分隔线和总结说明

---

## 2. 代码块规范

### 2.1 语言标签
- Rust 代码块：` ```rust `
- 幻语代码块：` ```hl `
- 汇编代码块：` ```gas `
- TOML：` ```toml `
- 正则表达式：` ```regex `
- EBNF：` ```ebnf `
- 通用文本：` ``` `

### 2.2 代码示例
- 示例代码应简洁、具有代表性
- 重要代码添加注释说明关键部分
- 避免过于冗长的示例代码

---

## 3. 表格规范

### 3.1 表格格式
- 使用对齐格式确保表格美观
- 表头居中对齐，内容左对齐

```markdown
| 属性 | 值 | 说明 |
|:-----|:-----|:-----|
| 名称 | 示例 | 详细说明 |
```

---

## 4. 文字规范

### 4.1 语言一致性
- 使用规范的中文表述
- 技术术语统一
- 中英文混杂时适当留空格

### 4.2 强调方式
- **重要概念**：加粗
- `代码元素`：行内代码
- > 提示/说明：引用块
- *强调性文字*：斜体（谨慎使用）

### 4.3 列表规范
- 有序列表：使用数字编号
- 无序列表：使用 `-` 作为项目符号
- 每个列表项简洁明了

---

## 5. 章节编号规范

### 5.1 章节结构
```
第 N 章: [章节名称]
  N.1 [节名称]
    N.1.1 [子节名称]
```

### 5.2 链接格式
- 章节内部链接：`[文字描述](#锚点)`
- 跨文档链接：`[文字描述](其他文档.md)`

---

## 6. 版本信息

### 6.1 文档版本
- 版本号与项目版本同步 (v0.4.0)
- 更新日期每月刷新

---

## 7. 修订记录（可选）
重大修改可在文档末尾添加修订记录
```
## 修订记录
- v0.4.0 (2026-05): 初始版本
```

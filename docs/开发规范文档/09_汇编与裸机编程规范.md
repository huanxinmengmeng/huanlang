# 9. 汇编与裸机编程规范

## 9.1 汇编与裸机编程概述

幻语提供完整的底层汇编和裸机编程支持，使开发者能够编写从硬件初始化到应用程序的完整系统软件。

**核心能力**：
- **独立汇编文件**（`.hasm`）：支持 GNU 汇编器（GAS）语法，编写启动代码、中断向量表、高度优化的汇编例程
- **内联汇编**（`asm!` 宏）：在幻语函数中直接嵌入汇编指令，用于性能关键路径或特殊硬件操作
- **全架构支持**：x86/x86_64、ARM/AArch64、RISC-V、AVR、MSP430、Xtensa 等
- **裸机编程**：零依赖启动、链接器脚本、内存映射寄存器操作
- **特权指令**：支持系统寄存器读写、特权级别切换、内存屏障、缓存控制
- **外部汇编互操作**：遵守目标平台调用约定，与幻语/C 代码无缝互操作

## 9.2 文件格式

| 扩展名 | 类型 | 描述 |
|--------|------|------|
| `.hasm` | 独立汇编文件 | 纯汇编代码，采用 GNU Assembler（GAS）语法 |
| `.hl` | 幻语源码 | 可包含内联汇编块（`汇编!` 宏） |
| `.S` | 预处理汇编 | 兼容 GCC 预处理器的汇编文件（可被幻语构建系统直接使用） |

## 9.3 独立汇编模块（.hasm）

### 9.3.1 语法规范

独立汇编文件完全兼容 **GNU Assembler（GAS）** 语法，使用目标平台的指令集。幻语编译器将 `.hasm` 文件直接传递给系统汇编器（`as` 或 `llvm-mc`）处理。

**基本结构**：
```gas
.syntax unified
.cpu cortex-m4
.thumb

.section .vector_table, "a"
.global _start
.type _start, %function

_start:
    bl  main
loop:
    b   loop
```

### 9.3.2 汇编器指令（Directives）

| 指令 | 说明 | 示例 |
|------|------|------|
| `.syntax` | 设置汇编语法风格 | `.syntax unified` |
| `.cpu` | 指定目标 CPU | `.cpu cortex-m4` |
| `.thumb` / `.arm` | 切换指令集 | `.thumb` |
| `.section` | 定义段 | `.section .text, "ax"` |
| `.global` | 导出符号 | `.global _start` |
| `.type` | 指定符号类型 | `.type _start, %function` |
| `.size` | 指定符号大小 | `.size _start, . - _start` |
| `.word` / `.long` | 定义 32 位数据 | `.word 0x20001000` |
| `.byte` | 定义 8 位数据 | `.byte 0x01, 0x02` |
| `.ascii` / `.asciz` | 定义字符串 | `.asciz "Hello"` |
| `.align` | 对齐 | `.align 2` |
| `.macro` / `.endm` | 定义宏 | `.macro push reg \n str \reg, [sp, #-4]! \n .endm` |
| `.rept` / `.endr` | 重复块 | `.rept 10 \n nop \n .endr` |
| `.equ` / `.set` | 定义常量 | `.equ STACK_SIZE, 0x400` |
| `.include` | 包含其他文件 | `.include "registers.inc"` |

### 9.3.3 与幻语互操作约定

- **符号修饰**：幻语函数默认无名称修饰（与 C 相同）。使用 `.global` 导出汇编符号，可在幻语中通过 `外部` 块调用。
- **调用约定**：遵循目标平台的标准调用约定（如 ARM AAPCS、x86_64 System V）。
- **数据类型对应**：

| 幻语类型 | 汇编大小 | 说明 |
|---------|---------|------|
| `整数8` / `无符号8` | 1 字节 | `.byte` |
| `整数16` / `无符号16` | 2 字节 | `.hword` |
| `整数32` / `无符号32` | 4 字节 | `.word` |
| `整数64` / `无符号64` | 8 字节 | `.quad` |
| `指针[T]` | 4/8 字节 | 取决于架构 |

### 9.3.4 完整启动代码示例（Cortex-M4）

```gas
.syntax unified
.cpu cortex-m4
.thumb

.section .vector_table, "a"
.global _start

_vector_table:
    .word _stack_top
    .word _start
    .word nmi_handler
    .word hardfault_handler
    .word memmanage_handler
    .word busfault_handler
    .word usage_fault_handler
    .word 0, 0, 0, 0
    .word svc_handler
    .word debugmon_handler
    .word 0
    .word pendsv_handler
    .word systick_handler

.section .text
.type _start, %function
_start:
    ldr r0, =_sidata
    ldr r1, =_sdata
    ldr r2, =_edata
    bl  copy_data

    ldr r0, =_sbss
    ldr r1, =_ebss
    bl  zero_bss

    bl  main

loop:
    b   loop

.type copy_data, %function
copy_data:
    cmp r1, r2
    bge 2f
1:  ldr r3, [r0], #4
    str r3, [r1], #4
    cmp r1, r2
    blt 1b
2:  bx  lr

.type zero_bss, %function
zero_bss:
    mov r2, #0
    cmp r0, r1
    bge 2f
1:  str r2, [r0], #4
    cmp r0, r1
    blt 1b
2:  bx  lr

.weak nmi_handler
.weak hardfault_handler
.weak memmanage_handler
.weak busfault_handler
.weak usage_fault_handler
.weak svc_handler
.weak debugmon_handler
.weak pendsv_handler
.weak systick_handler

nmi_handler:
hardfault_handler:
memmanage_handler:
busfault_handler:
usage_fault_handler:
svc_handler:
debugmon_handler:
pendsv_handler:
systick_handler:
    b   .
```

## 9.4 内联汇编语法

### 9.4.1 基本语法

幻语内联汇编采用类似 Rust `asm!` 宏的语法设计：

```hl
汇编!(
    "汇编指令模板",
    输出(约束) 变量名,
    输入(约束) 变量名,
    破坏列表,
    选项...
)
```

### 9.4.2 完整语法定义（EBNF）

```ebnf
内联汇编 ::= "汇编!" "(" 
            模板字符串 { "," 模板字符串 }
             [ "," 输出操作数 { "," 输出操作数 } ]
             [ "," 输入操作数 { "," 输入操作数 } ]
             [ "," 破坏列表 ]
             [ "," 选项列表 ]
             ")"

模板字符串 ::= 字符串字面量

输出操作数 ::= "输出" "(" 约束 ")" 标识符
            | "输出" "(" 约束 ")" "_"
            | "输出" "(" 约束 ")" 标识符 "=>" 表达式

输入操作数 ::= "输入" "(" 约束 ")" 表达式

破坏列表 ::= "破坏" "(" 字符串 { "," 字符串 } ")"

选项列表 ::= 选项 { "," 选项 }
选项 ::= "纯" | "无内存" | "保留标志" | "不可达" | "对齐栈" | "英特尔语法" | "易失"
```

### 9.4.3 操作数约束

操作数约束遵循 LLVM/GCC 的约束码体系：

| 约束 | 中文名 | 拼音 | 说明 |
|------|--------|------|------|
| `r` | `寄存器` | `jicunqi` | 任意通用寄存器 |
| `m` | `内存` | `neicun` | 内存地址 |
| `i` | `立即数` | `lijishu` | 整数立即数 |
| `=r` | `输出寄存器` | `shuchu` | 输出到寄存器 |
| `+r` | `读写寄存器` | `duxie` | 输入输出使用同一寄存器 |
| `&r` | `早期破坏` | `zaoqipo` | 输出操作数不与输入共享寄存器 |
| `X` | `任意` | `renyi` | 任意类型的操作数 |

**架构特定约束**：
- **x86/x86_64**：`a` (eax/rax), `b` (ebx/rbx), `c` (ecx/rcx), `d` (edx/rdx), `S` (esi/rsi), `D` (edi/rdi)
- **ARM/AArch64**：`w` (VFP 浮点寄存器), `t` (Thumb 高寄存器 r8-r15)
- **RISC-V**：`f` (浮点寄存器)

### 9.4.4 破坏列表

| 破坏项 | 说明 |
|--------|------|
| `"寄存器名"` | 如 `"r0"`, `"rax"`，指定特定寄存器被修改 |
| `"cc"` | 条件码/标志寄存器被修改 |
| `"memory"` | 内存被写入，编译器需重新加载所有内存值 |

### 9.4.5 选项列表

| 选项 | 中文 | 说明 |
|------|------|------|
| `纯`/`pure` | `纯函数` | 汇编代码无副作用，不修改内存 |
| `无内存`/`nomem` | `无内存破坏` | 汇编代码不读写内存 |
| `保留标志`/`preserves_flags` | `保留标志` | 汇编代码不修改条件码 |
| `不可达`/`noreturn` | `不返回` | 汇编代码不返回（如跳转到重置向量） |
| `对齐栈`/`alignstack` | `对齐栈` | 汇编代码需要栈对齐 |
| `英特尔语法`/`intel_syntax` | `英特尔语法` | 使用 Intel 汇编语法（x86） |
| `易失`/`volatile` | `易失` | 禁止编译器优化重排汇编代码 |

### 9.4.6 内联汇编示例

```hl
-- 读取 x86_64 TSC 时间戳计数器
函数 读取时间戳() 返回 无符号64
开始
    令 低32位 类型 无符号32
    令 高32位 类型 无符号32
    
    汇编!(
        "rdtsc",
        输出("={eax}") 低32位,
        输出("={edx}") 高32位,
        选项("易失")
    )
    
    返回 (高32位 转为 无符号64) << 32 | (低32位 转为 无符号64)
结束

-- ARM 等待中断
函数 等待中断()
开始
    汇编!("wfi", 选项("易失"), 选项("无内存"))
结束

-- RISC-V 内存屏障
函数 内存屏障()
开始
    汇编!("fence iorw, iorw", 破坏("memory"))
结束

-- 使用 Intel 语法（x86）
函数 快速内存复制(目标 类型 指针[单元], 源 类型 指针[单元], 长度 类型 整数)
开始
    汇编!(
        ".intel_syntax noprefix",
        "rep movsb",
        ".att_syntax prefix",
        输入("{rdi}") 目标,
        输入("{rsi}") 源,
        输入("{rcx}") 长度,
        破坏("memory"),
        选项("英特尔语法")
    )
结束
```

## 9.5 多架构指令集支持

### 9.5.1 架构适配层

幻语编译器通过 **架构适配层（Architecture Adaptation Layer, AAL）** 支持多指令集：

```
┌─────────────────────────────────────────────────────────────┐
│                    幻语内联汇编块                            │
│            asm!("mov {0}, {1}", out(reg) x, in(reg) y)     │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                架构适配层（AAL）                             │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐          │
│  │  x86_64 │ │  ARM    │ │ RISC-V  │ │  AVR    │ ...      │
│  │ 适配器  │ │ 适配器  │ │ 适配器  │ │ 适配器  │          │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘          │
│       │           │           │           │                 │
│       ▼           ▼           ▼           ▼                 │
│  寄存器映射   调用约定    汇编方言    约束转换               │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│              LLVM 内联汇编 / 目标汇编器                      │
│         x86 (AT&T/Intel) | ARM UAL | RISC-V | ...           │
└─────────────────────────────────────────────────────────────┘
```

### 9.5.2 完整架构支持列表

| 架构 | 目标三元组 | 汇编方言 | 主要应用场景 |
|------|-----------|---------|-------------|
| x86 | `i386-*` | AT&T / Intel | 传统 PC、嵌入式 x86 |
| x86_64 | `x86_64-*` | AT&T / Intel | 服务器、桌面、高性能计算 |
| ARM (32-bit) | `arm-*`, `thumb-*` | UAL | 嵌入式、移动设备 |
| AArch64 | `aarch64-*` | ARMv8-A 汇编 | 服务器、高端移动设备 |
| RISC-V (32) | `riscv32-*` | RISC-V 汇编 | 嵌入式、IoT |
| RISC-V (64) | `riscv64-*` | RISC-V 汇编 | 服务器、高性能计算 |
| MIPS | `mips-*` | MIPS 汇编 | 路由器、嵌入式 |
| PowerPC | `powerpc-*` | PowerPC 汇编 | 工控、汽车电子 |
| AVR | `avr-*` | AVR 汇编 | Arduino、微控制器 |
| MSP430 | `msp430-*` | MSP430 汇编 | 超低功耗嵌入式 |
| Xtensa | `xtensa-*` | Xtensa 汇编 | ESP32、ESP8266 |
| WebAssembly | `wasm32-*` | WAT / Wasm 字节码 | 浏览器、边缘计算 |

### 9.5.3 架构特定约束验证

```rust
pub fn is_valid_constraint_for_type(constraint: &str, ty: &Type, arch: Arch) -> bool {
    match arch {
        Arch::X86_64 => {
            match constraint {
                "r" => matches!(ty, Type::Int | Type::I64 | Type::Ptr(_)),
                "a" | "b" | "c" | "d" | "S" | "D" => matches!(ty, Type::Int | Type::I64),
                "x" | "y" | "z" => matches!(ty, Type::F32 | Type::F64),
                "m" => true,
                "i" => true,
                _ => false,
            }
        }
        Arch::ARM => {
            match constraint {
                "r" => matches!(ty, Type::Int | Type::I32 | Type::Ptr(_)),
                "w" => matches!(ty, Type::F32 | Type::F64),
                "t" => matches!(ty, Type::Int | Type::I32),
                "m" => true,
                "i" => true,
                _ => false,
            }
        }
        Arch::RiscV => {
            match constraint {
                "r" => matches!(ty, Type::Int | Type::I64 | Type::Ptr(_)),
                "f" => matches!(ty, Type::F32 | Type::F64),
                "m" => true,
                "i" => true,
                _ => false,
            }
        }
        _ => true, // 保守处理
    }
}
```

## 9.6 寄存器操作与约束规范

### 9.6.1 统一寄存器命名

| 抽象名 | x86_64 | ARM | RISC-V | 说明 |

| `帧指针` | `rbp` | `fp` | `fp/s0` | 帧指针 |
| `链接寄存器` | — | `lr` | `ra` | 返回地址 |
| `程序计数器` | `rip` | `pc` | `pc` | 程序计数器 |

### 9.6.2 系统寄存器访问

```hl

令 当前程序地址 为 读取寄存器(程序计数器)

-- 写入系统寄存器
写入寄存器(栈指针, 新栈顶)

-- ARM 特定系统寄存器
```

## 9.7 中断与特权指令

### 9.7.1 中断处理语法

幻语提供专用的中断处理语法，确保正确的寄存器保存和恢复：

```hl
-- 定义中断服务函数
中断 定时器中断:
    保存上下文
    // 中断处理代码
    清除中断标志()
    恢复上下文
    中断返回
结束

-- 启用/禁用中断
启用中断()
禁用中断()

-- 设置中断优先级
设置中断优先级(定时器向量, 优先级设为 2)
```

### 9.7.2 中断服务函数属性

```hl
@中断
@调用约定("中断")
函数 定时器2_中断处理器()
开始
    令 标志 为 定时器2.状态.位(0)
    若 标志 为 真 则
        处理定时事件()
        定时器2.状态.位(0) 设为 假
    结束
结束
```

编译器自动生成中断序言和尾声，保存和恢复所有调用者保存的寄存器，并使用正确的返回指令（如 ARM 的 `bx lr` 或 x86 的 `iretq`）。

```hl
汇编!("svc #0")  -- ARM 系统调用
汇编!("syscall")  -- x86_64 系统调用

```

## 9.8 外设寄存器定义

### 9.8.1 寄存器定义语法

```hl
外设 GPIOA 基址 0x40010800:
    寄存器 CRL  偏移 0x00, 类型 无符号32, 访问 读写
    寄存器 CRH  偏移 0x04, 类型 无符号32, 访问 读写
    寄存器 IDR  偏移 0x08, 类型 无符号32, 访问 只读
    寄存器 ODR  偏移 0x0C, 类型 无符号32, 访问 读写
    寄存器 BSRR 偏移 0x10, 类型 无符号32, 访问 只写
    寄存器 BRR  偏移 0x14, 类型 无符号32, 访问 只写
    寄存器 LCKR 偏移 0x18, 类型 无符号32, 访问 读写
结束
```

### 9.8.2 寄存器访问操作

```hl
-- 位操作
令 GPIOA.CRL.位域(0..4) 设为 0b0011  -- 设置 4 位域
令 引脚状态 为 GPIOA.IDR.位(0)        -- 读取单一位

-- 直接赋值
GPIOA.BSRR 设为 (1 << 0)              -- 设置 PA0
GPIOA.BRR 设为 (1 << 0)               -- 清除 PA0

-- 使用枚举或常量
GPIOA.CRL 设为 GPIO模式.推挽输出 | GPIO速度.50MHz
```

### 9.8.3 编译器处理

编译器将寄存器访问转换为 `volatile` 指针解引用，确保每次访问都生成实际的内存读写指令，不被优化或重排。

```rust
// 内部生成代码
const GPIOA_BASE: usize = 0x40010800;
const GPIOA_BSRR: *mut u32 = (GPIOA_BASE + 0x10) as *mut u32;
unsafe { core::ptr::write_volatile(GPIOA_BSRR, 1 << 0); }
```

## 9.9 链接器脚本与内存布局

### 9.9.1 幻语风格链接器脚本

幻语提供一种声明式的链接器脚本语法，编译时生成目标平台的 ld 脚本：

```hl
内存布局:
    闪存: 起始 0x08000000, 长度 64K, 属性(可读, 可执行)
    内存: 起始 0x20000000, 长度 20K, 属性(可读, 可写, 可执行)

段定义:
    .向量表: 放入 闪存, 对齐 256
        包含 向量表数据
    
    .文本: 放入 闪存
        包含 代码段, 只读数据
    
    .数据: 放入 内存, 加载地址在 闪存
        包含 已初始化数据
    
    .bss: 放入 内存
        包含 未初始化数据
        清零
    
    .堆: 放入 内存
        大小 4K
    
    .栈: 放入 内存
        大小 2K
        地址 内存结束 - 2K
结束
```

### 9.9.2 生成的传统 LD 脚本

```ld
MEMORY
{
    FLASH (rx) : ORIGIN = 0x08000000, LENGTH = 64K
    RAM (rwx)  : ORIGIN = 0x20000000, LENGTH = 20K
}

SECTIONS
{
    .vector_table : ALIGN(256) { KEEP(*(.vector_table)) } > FLASH
    .text : { *(.text*) *(.rodata*) } > FLASH
    .data : { *(.data*) } > RAM AT> FLASH
    .bss : { *(.bss*) } > RAM
    _sidata = LOADADDR(.data);
    _sdata = ADDR(.data);
    _edata = ADDR(.data) + SIZEOF(.data);
    _sbss = ADDR(.bss);
    _ebss = ADDR(.bss) + SIZEOF(.bss);
    _stack_top = ORIGIN(RAM) + LENGTH(RAM);
}
```

### 9.11.2 编译器预定义宏

| 宏 | 说明 |
|-----|------|
| `架构_x86` | x86 架构 |
| `架构_x86_64` | x86_64 架构 |
| `架构_arm` | ARM 32位架构 |
| `架构_aarch64` | AArch64 架构 |
| `架构_riscv32` | RISC-V 32位 |
| `架构_riscv64` | RISC-V 64位 |
| `架构_avr` | AVR 架构 |
| `裸机` | 裸机环境（无操作系统） |
| `有操作系统` | 运行在操作系统之上 |
| `芯片_` + 芯片型号 | 如 `芯片_stm32f103` |

**条件编译示例**：
```hl
@若 架构_arm
    汇编!("wfi")  -- ARM 等待中断
@否则若 架构_x86_64
    汇编!("hlt")  -- x86 停机
@结束
```

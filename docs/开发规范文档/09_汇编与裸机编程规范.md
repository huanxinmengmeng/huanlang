## 9.1 汇编与裸机编程概述

幻语提供完整的底层汇编和裸机编程支持，使开发者能够编写从硬件初始化到应用程序的完整系统软件。语言内置了符合现代安全实践的内联汇编机制，同时保留了使用独立汇编文件编写启动代码、中断服务例程的能力。

核心能力：
- **独立汇编文件**（`.hasm`）：支持 GNU 汇编器（GAS）语法，编写启动代码、中断向量表、高度优化的汇编例程。
- **内联汇编**（`asm!` 宏）：在幻语函数中直接嵌入汇编指令，用于性能关键路径或特殊硬件操作。
- **全架构支持**：x86/x86_64、ARM/AArch64、RISC‑V、AVR、MSP430、Xtensa 等。
- **裸机编程**：零依赖启动、链接器脚本、内存映射寄存器操作。
- **特权指令**：支持系统寄存器读写、特权级别切换、内存屏障、缓存控制。
- **外部汇编互操作**：遵守目标平台调用约定，与幻语/C 代码无缝互操作。

## 9.2 文件格式与扩展名

| 扩展名 | 类型 | 描述 |
|--------|------|------|
| `.hasm` | 独立汇编文件 | 纯汇编代码，采用 GNU Assembler（GAS）语法 |
| `.hl`  | 幻语源码 | 可包含内联汇编块（`汇编!` 宏） |
| `.S`   | 预处理汇编 | 兼容 GCC 预处理器的汇编文件（可被幻语构建系统直接使用） |

## 9.3 内联汇编语法

### 9.3.1 基本形式

幻语内联汇编采用类似 Rust `asm!` 宏的语法，格式如下：

```
汇编!(
    "汇编指令模板",
    输出(约束) 变量名,
    输入(约束) 变量名,
    破坏列表,
    选项...
)
```

示例：
```hl
函数 读取时间戳() -> u64
开始
    汇编!(
        "rdtsc",
        输出("={eax}") 低32位,
        输出("={edx}") 高32位,
        选项("易失")
    )
    返回 (高32位 转为 u64) << 32 | (低32位 转为 u64)
结束
```

### 9.3.2 操作数约束

操作数约束沿用 LLVM/GCC 约束码体系，部分常用约束如下：

| 约束 | 中文名 | 说明 |
|------|--------|------|
| `r` | 寄存器 | 任意通用寄存器 |
| `m` | 内存 | 内存地址 |
| `i` | 立即数 | 整数立即数 |
| `=r` | 输出寄存器 | 输出到寄存器 |
| `+r` | 读写寄存器 | 输入输出使用同一寄存器 |
| `&r` | 早期破坏 | 输出操作数不与输入共享寄存器 |
| `X` | 任意 | 任意类型的操作数 |

架构特定约束：x86(`a`, `b`, `c`, `d`, `S`, `D`)，ARM(`w` 浮点寄存器，`t` 高位寄存器)，RISC‑V(`f` 浮点寄存器)。

### 9.3.3 破坏列表与选项

破坏列表告知编译器哪些资源被汇编代码修改：`"寄存器名"`、`"cc"`、`"memory"`。

可用选项：`纯`/`pure`（无副作用）、`无内存`/`nomem`（不读写内存）、`保留标志`/`preserves_flags`、`不可达`/`noreturn`、`对齐栈`/`alignstack`、`英特尔语法`/`intel_syntax`、`易失`/`volatile`。

## 9.4 多架构支持

编译器通过**架构适配层（AAL）**根据目标三元组选择正确的汇编器方言和约束映射。

支持的主要架构：
- x86_64 (AT&T/Intel)
- ARM (UAL), AArch64
- RISC‑V (32/64)
- AVR, MSP430, Xtensa (ESP32)

系统提供统一的寄存器抽象名，例如 `通用寄存器0` 在不同架构下映射到 `rax`/`r0`/`a0`。

## 9.5 中断与特权指令

为中断服务函数提供专用属性和语法：

```hl
@中断
@调用约定("中断")
函数 定时器2中断处理器()
开始
    令 标志 = 定时器2.状态.位(0)
    若 标志 为 真 则
        处理定时事件()
        定时器2.状态.位(0) 设为 假
    结束
结束
```

编译器自动生成上下文保存/恢复代码，并使用正确的返回指令（如 ARM `bx lr` 或 RISC‑V `mret`）。

通过内联汇编访问特权寄存器：`汇编!("mrs {0}, cpsr", 输出(reg) cpsr值)`。

## 9.6 外设寄存器定义

提供声明式的寄存器块语法，最终生成 `volatile` 指针操作：

```hl
外设 GPIOA 基址 0x40010800:
    寄存器 CRL  偏移 0x00, 类型 u32, 访问 读写
    寄存器 BSRR 偏移 0x10, 类型 u32, 访问 只写
结束

// 使用
GPIOA.CRL.位域(0..4) 设为 0b0011
GPIOA.BSRR 设为 (1 << 0)
```

支持位操作和位域操作，编译器负责保证每次访问都是 `volatile` 的。

## 9.7 链接器脚本与裸机项目

幻语提供声明式的链接脚本语法（`.hld`），可自动生成目标 LD 脚本：

```hl
内存布局:
    闪存: 起始 0x08000000, 长度 64K, 属性(可读,可执行)
    内存: 起始 0x20000000, 长度 20K, 属性(可读,可写,可执行)

段定义:
    .向量表: 放入 闪存, 对齐 256
    .文本:   放入 闪存
    .数据:   放入 内存, 加载地址在 闪存
    .bss:    放入 内存, 清零
```

裸机项目模板通过 `huan new --template bare` 创建，包含 `启动代码.has m`、`主程序.hl`、链接脚本和目标配置。

## 9.8 工具链与编译命令

新增命令：
- `huan asm <file> --target <arch>`：汇编独立文件
- `huan flash <firmware> --chip <mcu> --interface <swd/jtag>`：烧录
- `huan debug --remote :3333`：远程调试

编译裸机程序：
```bash
huan asm 启动代码.has m --target thumbv7em-none-eabi -o 启动代码.o
huan build 主程序.hl --target thumbv7em-none-eabi -O z --link-script 链接脚本.hld
huan flash 闪烁.bin --chip STM32F103C8 --interface swd
```

编译器预定义一系列宏用于条件编译，如 `架构_arm`、`裸机`、`芯片_stm32f103`。

## 9.9 完整示例：Cortex‑M3 LED 闪烁

```hl
@baremetal
外设 GPIOC 基址 0x40011000:
    寄存器 CRH  偏移 0x04, 类型 u32
    寄存器 BSRR 偏移 0x10, 类型 u32
结束
外设 RCC 基址 0x40021000:
    寄存器 APB2ENR 偏移 0x18, 类型 u32
结束

@导出 "C"
函数 主()
开始
    令 RCC.APB2ENR.位(4) 设为 真          // 使能 GPIOC 时钟
    令 GPIOC.CRH.位域(20..24) 设为 0b0011 // PC13 推挽输出

    当 真 循环
        GPIOC.BSRR 设为 (1 << 13)         // 亮
        延迟(500000)
        GPIOC.BSRR 设为 (1 << (13+16))    // 灭
        延迟(500000)
    结束
结束

函数 延迟(计数: u32)
开始
    重复 计数 次
        汇编!("nop")
    结束
结束
```

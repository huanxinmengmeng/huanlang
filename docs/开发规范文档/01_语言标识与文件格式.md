# 1. 语言标识与文件格式

## 1.1 语言标识符

| 属性 | 值 |
|------|-----|
| 语言名称 | 幻语（HuanLang） |
| 版本 | 0.0.1 |
| 文件扩展名 | `.hl`（源代码）、`.hla`（AI交换格式）、`.hlm`（模块接口）、`.hlp`（包文件）、`.hasm`（独立汇编） |
| MIME 类型 | `text/x-huan` |
| 编码 | UTF-8 无 BOM |
| 换行符 | LF（`\n`）推荐，CRLF 兼容 |

**语言标识符常量（用于编译器内部）**：
```rust
pub const LANGUAGE_NAME: &str = "HuanLang";
pub const LANGUAGE_VERSION: &str = "0.0.1";
pub const FILE_EXTENSIONS: &[&str] = &["hl", "hla", "hlm", "hlp", "hasm"];
```

## 1.2 文件格式详细规范

### 1.2.1 `.hl` 源代码文件

**文件结构（EBNF）**：
```ebnf
hl文件 ::= [ Shebang行 ] { 顶层元素 }
Shebang行 ::= "#!" { 任意字符 } "\n"
顶层元素 ::= 导入语句 | 模块定义 | 函数定义 | 结构定义 | 特征定义 | 实现块 | 全局变量 | 外部块 | 类型别名
```

**Shebang 处理规范**：
- 若文件首两个字节为 `#!`，编译器应跳过第一行。
- Shebang 行不计入源码位置的行号计算（即源码第1行为 Shebang 后的第一行）。
- Shebang 行支持参数传递，例如 `#!/usr/bin/env huan --ownership`，参数应传递给编译器。

**示例**：
```hl
#!/usr/bin/env huan --ownership

导入 "标准库/核心"

函数 主() 返回 整数
开始
    显示("你好，幻语！")
    返回 0
结束
```

### 1.2.2 `.hla` AI 交换格式文件

**设计目标**：
- 每行一个操作，便于逐行流式解析。
- 无歧义、无嵌套，适合 AI 模型生成。
- 标签可选，用于跳转目标。

**文件格式规范**：
```ebnf
hla文件 ::= [ 元数据行 ] { 操作行 | 注释行 | 空行 }
元数据行 ::= "#!" 键 "=" 值
操作行 ::= [ 标签 ] 操作码 操作数...
标签 ::= "L" 数字
注释行 ::= "#" { 任意字符 }
空行 ::= "\n"
```

**元数据字段**：
| 键 | 说明 | 示例 |
|----|------|------|
| `版本` | 幻语版本 | `#! 版本 = "1.2"` |
| `来源` | 生成来源 | `#! 来源 = "transpiler"` |
| `关键词风格` | 原始关键词风格 | `#! 关键词风格 = "中文"` |
| `目标` | 目标平台 | `#! 目标 = "wasm32"` |

**示例**：
```
#! 版本 = "1.2"
#! 来源 = "huan-compiler"
#! 关键词风格 = "中文"

# 计算斐波那契数列
L001 HANSHU 斐波那契 参数:整数 返回:整数
L002   RUO 参数 小于 2
L003     FANHUI 参数
L004   JIESHU
L005   JIAN 减一 参数 1
L006   DIAOYONG 结果一 斐波那契 减一
L007   JIAN 减二 参数 2
L008   DIAOYONG 结果二 斐波那契 减二
L009   JIA 结果 结果一 结果二
L010   FANHUI 结果
L011 JIESHU
```

### 1.2.3 `.hasm` 独立汇编文件

**用途**：
- 裸机启动代码（中断向量表、栈初始化）
- 高度优化的汇编例程
- 调用特定架构的特殊指令

**支持语法**：
- GNU Assembler（GAS）语法
- 支持 `.macro`、`.rept`、`.irp` 等汇编宏
- 支持条件汇编 `.if`、`.else`、`.endif`

**与幻语互操作约定**：
- 幻语函数默认名称修饰规则：无修饰（与 C 相同）。
- 使用 `.global` 导出符号供幻语调用。
- 调用约定遵循目标平台标准。

**示例**：
```gas
.syntax unified
.cpu cortex-m4
.thumb

.section .vector_table, "a"
.global _start
.type _start, %function

_start:
    bl  main
loop:
    b   loop
```

### 1.2.4 `.hlm` 模块接口文件

**用途**：
- 定义模块的公开 API，供编译器和 IDE 使用。
- 不包含实现代码，仅包含函数签名、类型定义、常量。

**语法**：与幻语源文件类似，但函数体被省略（以分号结尾）。

**示例**：
```hl
公开 函数 计算(甲: 整数, 乙: 整数) -> 整数;
公开 结构 点
    字段 横坐标: 浮点64
    字段 纵坐标: 浮点64
结束
公开 常数 PI: 浮点64 = 3.141592653589793;
```

### 1.2.5 `.hlp` 包文件

**格式**：tar.gz 压缩包，内部结构如下：

```
包名-版本/
├── 幻语包.toml         # 包描述文件（必需）
├── 源码/               # 源代码目录
│   ├── 库.hl
│   └── ...
├── 示例/               # 示例代码（可选）
├── 测试/               # 测试代码（可选）
├── 文档/               # 文档（可选）
├── LICENSE             # 许可证文件
└── README.md           # 自述文件
```

**幻语包.toml 格式**：
```toml
[package]
name = "网络工具"
version = "0.3.0"
authors = ["张三 <zhangsan@example.com>"]
description = "幻语网络工具库"
license = "MIT"
repository = "https://gitee.com/example/network-tools"
keyword_style = "Chinese"

[dependencies]
序列化 = "1.0"

[extern]
c_libs = ["curl"]
```

## 1.3 编码与字符处理规范

### 1.3.1 UTF-8 编码要求

- 所有幻语源文件必须使用 **UTF-8 编码，且不带 BOM**。
- 编译器若检测到 UTF-8 BOM（`EF BB BF`），应发出警告并自动跳过。
- 非 UTF-8 编码文件应报错并终止编译。

### 1.3.2 行结束符规范化

- 编译器在词法分析前将所有 `\r\n` 和 `\r` 统一转换为 `\n`。
- 转换后，源码位置的行号基于 `\n` 计数。
- 原始文件中的 `\r\n` 在错误报告中的列号应正确反映（按 UTF-8 字符计数）。

### 1.3.3 标识符字符集

**允许的字符**：
- ASCII 字母：`A-Z a-z`
- 数字：`0-9`（不能作为首字符）
- 下划线：`_`
- CJK 统一表意字符：Unicode 范围见下表

| 区块 | 范围 | 说明 |
|------|------|------|
| CJK 统一表意字符 | `U+4E00` - `U+9FFF` | 常用汉字 |
| CJK 扩展 A | `U+3400` - `U+4DBF` | 罕用汉字 |
| CJK 扩展 B | `U+20000` - `U+2A6DF` | 古汉字、方言字 |
| CJK 兼容表意字符 | `U+F900` - `U+FAFF` | 兼容字符 |

**标识符正则表达式**：
```regex
[\p{Unified_Ideograph}a-zA-Z_][\p{Unified_Ideograph}a-zA-Z0-9_]*
```

**注意**：
- 标识符最大长度：255 字符。
- 标识符内部比较采用 Unicode 规范化形式 C（NFC）。

### 1.3.4 注释与空白

- 空白字符包括：空格（`U+0020`）、制表符（`U+0009`）、换行（`U+000A`）、回车（`U+000D`）。
- 单行注释：`--` 开头，至行尾。
- 多行注释：`/* ... */`，支持嵌套。

## 1.4 文件魔术数字与识别

| 文件类型 | 魔术数字（十六进制） | 说明 |
|---------|---------------------|------|
| `.hl` | `23 21`（可选） | 若以 `#!` 开头，则为 Shebang |
| `.hla` | `23 21`（可选） | 若以 `#!` 开头，则为元数据 |
| `.hlp` | `1F 8B` | gzip 压缩文件标志 |
| `.hasm` | 无固定魔术数字 | 纯文本汇编 |

**编译器识别逻辑**：
1. 根据文件扩展名选择解析器。
2. 对于 `.hl` 和 `.hla`，若内容以 `#!` 开头，跳过第一行后继续解析。
3. 对于 `.hlp`，解压 tar.gz 后读取 `幻语包.toml`。

## 1.5 文件路径与模块解析

### 1.5.1 模块搜索路径

编译器按以下顺序搜索导入的模块：
1. 当前源文件所在目录。
2. `HUAN_PATH` 环境变量指定的目录。
3. 系统包缓存目录（`~/.huan/注册表/缓存/`）。
4. 标准库目录（编译器内置）。

### 1.5.2 模块名到文件路径的映射

| 导入语句 | 搜索路径 |
|---------|---------|
| `导入 "网络"` | `网络.hl`、`网络/库.hl`、`网络.hlm` |
| `导入 "幻语/集合"` | `$HUAN_PATH/幻语/集合.hl`、`标准库/集合.hl` |
| `导入 { 列表 } 从 "集合"` | 同上 |

## 1.6 编译器内部表示

### 1.6.1 源文件表示结构

```rust
pub struct SourceFile {
    pub path: PathBuf,
    pub content: String,
    pub lines: Vec<usize>,  // 每行起始字节偏移
    pub shebang: Option<String>,
}

impl SourceFile {
    pub fn new(path: PathBuf, content: String) -> Self {
        let (shebang, content) = Self::strip_shebang(content);
        let lines = Self::compute_line_offsets(&content);
        Self { path, content, lines, shebang }
    }
    
    pub fn position_to_offset(&self, line: usize, column: usize) -> usize {
        self.lines[line.saturating_sub(1)] + column.saturating_sub(1)
    }
    
    pub fn offset_to_position(&self, offset: usize) -> (usize, usize) {
        let line = self.lines.binary_search(&offset).unwrap_or_else(|e| e - 1);
        let column = offset - self.lines[line] + 1;
        (line + 1, column)
    }
}
```

### 1.6.2 文件标识符（FileId）

编译器使用 `FileId` 唯一标识每个源文件，用于错误报告和调试信息。

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct FileId(u32);

pub struct SourceManager {
    files: Vec<SourceFile>,
    file_ids: HashMap<PathBuf, FileId>,
}

impl SourceManager {
    pub fn add_file(&mut self, path: PathBuf, content: String) -> FileId;
    pub fn get_file(&self, id: FileId) -> &SourceFile;
}
```

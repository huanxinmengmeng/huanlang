# 2. 三语关键词系统

## 2.1 设计原则

幻语同时支持**中文**、**拼音**和**英文**三种关键词体系。编译器内部统一映射到相同的 `TokenKind`，三种写法可以**自由混用**。这一设计使得开发者可以根据自己的习惯选择关键词语言，降低了学习曲线，同时为国际化协作和 AI 代码生成提供了灵活性。

**核心原则**：
1. **语义等价**：三种语言的关键词在语法和语义上完全等价。
2. **任意混用**：同一源文件中可以混合使用三种语言的关键词。
3. **统一内部表示**：编译器在词法分析阶段将所有三语关键词转换为统一的 `TokenKind`。
4. **双向转换**：提供工具将源代码的关键词风格统一转换为指定语言。

## 2.3 词法分析中的关键词识别

### 2.3.1 关键词映射表数据结构

```rust
pub struct KeywordTable {
    // 字符串 -> TokenKind 的映射
    map: HashMap<&'static str, TokenKind>,
    // 反向映射：TokenKind -> 中文
    to_chinese: HashMap<TokenKind, &'static str>,
    // TokenKind -> 拼音
    to_pinyin: HashMap<TokenKind, &'static str>,
    // TokenKind -> 英文
    to_english: HashMap<TokenKind, &'static str>,
}

impl KeywordTable {
    pub fn new() -> Self {
        let mut map = HashMap::new();
        let mut to_chinese = HashMap::new();
        let mut to_pinyin = HashMap::new();
        let mut to_english = HashMap::new();
        
        // 插入三语映射
        for (ch, py, en, kind) in KEYWORD_TRIPLES {
            map.insert(ch, kind.clone());
            map.insert(py, kind.clone());
            map.insert(en, kind.clone());
            to_chinese.insert(kind.clone(), ch);
            to_pinyin.insert(kind.clone(), py);
            to_english.insert(kind.clone(), en);
        }
        // 类型关键词同理
        // ...
        
        Self { map, to_chinese, to_pinyin, to_english }
    }
    
    pub fn get(&self, s: &str) -> Option<TokenKind> {
        self.map.get(s).cloned()
    }
    
    pub fn to_style(&self, kind: &TokenKind, style: KeywordStyle) -> &'static str {
        match style {
            KeywordStyle::Chinese => self.to_chinese.get(kind).copied().unwrap_or(""),
            KeywordStyle::Pinyin => self.to_pinyin.get(kind).copied().unwrap_or(""),
            KeywordStyle::English => self.to_english.get(kind).copied().unwrap_or(""),
        }
    }
}
```

### 2.3.2 关键词三语元组定义

```rust
const KEYWORD_TRIPLES: &[(&str, &str, &str, TokenKind)] = &[

    ("为", "wei", "be", TokenKind::Be),

```

### 2.3.3 词法分析器集成

```rust
impl<'a> Lexer<'a> {
    fn lex_identifier_or_keyword(&mut self, start: SourcePosition) -> Result<Token, LexError> {
        let mut lexeme = String::new();
        while let Some(ch) = self.current_char() {
            if is_identifier_char(ch) {
                lexeme.push(ch);
                self.advance();
            } else {
                break;
            }
        }
        let end = self.current_position();
        let span = SourceSpan { start, end };
        
        // 查询关键词表
        let kind = self.keyword_table.get(&lexeme)
            .unwrap_or_else(|| TokenKind::Ident(lexeme.clone()));
            
        Ok(Token { kind, span, lexeme })
    }
}
```

## 2.4 关键词风格转换工具

### 2.4.1 命令行接口

```bash
huan fmt <输入文件> --style <chinese|pinyin|english> [--output <输出文件>]
```

### 2.4.2 转换算法

```rust
pub struct KeywordStyleConverter {
    table: KeywordTable,
}

impl KeywordStyleConverter {
    pub fn convert(&self, tokens: &[Token], style: KeywordStyle) -> String {
        let mut output = String::new();
        for token in tokens {
            match &token.kind {
                // 关键词：替换为目标风格
                TokenKind::Let | TokenKind::Const | TokenKind::If | TokenKind::Else | ... => {
                    output.push_str(self.table.to_style(&token.kind, style));
                }
                // 类型关键词：同样替换
                TokenKind::TypeInt | TokenKind::TypeI32 | ... => {
                    output.push_str(self.table.to_style(&token.kind, style));
                }
                // 标识符、字面量、符号：保持原样
                _ => output.push_str(&token.lexeme),
            }
            // 保留原始空白（简化处理，实际应基于 Span 重建）
        }
        output
    }
}
```

### 2.4.3 基于 AST 的精确转换

为提高转换质量，应基于 AST 而非 Token 流进行转换，以便正确处理注释和空白。

```rust
impl KeywordStyleConverter {
    pub fn convert_ast(&self, program: &Program, style: KeywordStyle) -> String {
        let mut emitter = CodeEmitter::new(style, &self.table);
        emitter.emit_program(program)
    }
}

struct CodeEmitter {
    style: KeywordStyle,
    table: KeywordTable,
    output: String,
}

impl CodeEmitter {
    fn emit_token(&mut self, kind: &TokenKind) {
        if let Some(keyword) = self.table.try_to_style(kind, self.style) {
            self.output.push_str(keyword);
        } else {
            // 非关键词，无法处理，使用默认
            self.output.push_str(&format!("{:?}", kind));
        }
    }
    
    fn emit_item(&mut self, item: &Item) {
        match item {
            Item::Function(func) => {
                if func.public {
                    self.emit_token(&TokenKind::Pub);
                    self.output.push(' ');
                }
                self.emit_token(&TokenKind::Func);
                self.output.push(' ');
                self.output.push_str(&func.name.name);
                // ... 参数、返回类型、函数体
            }
            // ...
        }
    }
}
```

## 2.6 关键词冲突处理

由于三语关键词共享相同的内部表示，以下情形不会产生歧义：
- 标识符 `let`（英文关键词）与英文关键词 `let` 冲突：词法分析器优先识别为关键词，因此不能将 `let` 用作标识符。若需要，可使用中文标识符 `令` 或拼音 `ling` 作为变量名（因为它们被识别为关键词）。
- 关键词与标识符在内部表示上明确分离（`TokenKind::Let` 与 `TokenKind::Ident("let".to_string())`）。

**编译器警告**：
当检测到标识符与三语关键词之一拼写相同时，应发出警告：
```
警告：标识符 "let" 与英文关键词冲突，建议使用其他名称
```

## 2.7 扩展性设计

### 2.7.1 新增关键词的步骤

1. 在 `TokenKind` 枚举中添加新变体。
2. 在 `KEYWORD_TRIPLES` 中添加对应的三语映射。
3. 更新语法解析器以处理新关键词。
4. 在 `KeywordStyleConverter` 中添加转换规则。

### 2.7.2 支持更多语言

幻语的三语系统可扩展支持其他语言（如日语、韩语）。只需在关键词表中添加新语言列，并扩展 `KeywordStyle` 枚举。

```rust
pub enum KeywordStyle {
    Chinese,
    Pinyin,
    English,
    Japanese,  // 未来扩展
    Korean,    // 未来扩展
}
```

---

## 2.1 设计目标

幻语的三语关键词系统旨在实现以下目标：

1. **语义等价**：中文、拼音、英文三种形式的关键词在语法和语义上完全等价，编译器内部统一处理。
2. **自由混写**：同一源文件中可混合使用三种语言的关键词，词法分析器自动识别并归一化。
3. **统一内部表示**：所有三语形式在词法分析阶段映射到相同的 `TokenKind`，后续编译阶段无需关心原始语言。
4. **双向转换**：提供工具将源代码的关键词风格统一转换为指定语言，且转换过程不改变代码语义。
5. **可扩展性**：关键词表设计预留扩展空间，未来可支持更多语言（如日语、韩语）。

## 2.2 关键词分类

幻语的关键词按语义分为以下类别：

| 类别 | 中文示例 | 英文示例 | 说明 |
|:-----|:---------|:---------|:-----|
| 变量/常量声明 | `令`、`变量`、`定`、`常量`、`为` | `let`、`const`、`be` | 声明变量或常量 |
| 类型标注 | `类型` | `type` | 显式类型标注 |
| 控制流 | `若`、`如果`、`则`、`那么`、`否则`、`或者`、`结束` | `if`、`then`、`else`、`end` | 条件分支 |

## 2.4 词法分析中的关键词识别

### 2.4.1 关键词映射表数据结构

```rust
/// 三语关键词映射表
pub struct KeywordTable {
    /// 字符串 → TokenKind 的正向映射
    map: HashMap&lt;&amp;'static str, TokenKind&gt;,
    /// TokenKind → 中文 的反向映射
    to_chinese: HashMap&lt;TokenKind, &amp;'static str&gt;,
    /// TokenKind → 拼音 的反向映射
    to_pinyin: HashMap&lt;TokenKind, &amp;'static str&gt;,
    /// TokenKind → 英文 的反向映射
    to_english: HashMap&lt;TokenKind, &amp;'static str&gt;,
}
```

### 2.4.2 构建函数

```rust
impl KeywordTable {
    pub fn new() -&gt; Self {
        let mut map = HashMap::new();
        let mut to_chinese = HashMap::new();
        let mut to_pinyin = HashMap::new();
        let mut to_english = HashMap::new();

        // 插入三语关键词元组
        for (ch, py, en, kind) in KEYWORD_TRIPLES.iter() {
            map.insert(ch, kind.clone());
            map.insert(py, kind.clone());
            map.insert(en, kind.clone());
            to_chinese.insert(kind.clone(), ch);
            to_pinyin.insert(kind.clone(), py);
            to_english.insert(kind.clone(), en);
        }

        Self { map, to_chinese, to_pinyin, to_english }
    }

    /// 根据字符串查找 TokenKind
    pub fn get(&amp;self, s: &amp;str) -&gt; Option&lt;TokenKind&gt; {
        self.map.get(s).cloned()
    }

    /// 根据 TokenKind 获取指定风格的关键词字符串
    pub fn to_style(&amp;self, kind: &amp;TokenKind, style: KeywordStyle) -&gt; &amp;'static str {
        match style {
            KeywordStyle::Chinese =&gt; self.to_chinese.get(kind).copied().unwrap_or(""),
            KeywordStyle::Pinyin =&gt; self.to_pinyin.get(kind).copied().unwrap_or(""),
            KeywordStyle::English =&gt; self.to_english.get(kind).copied().unwrap_or(""),
        }
    }
}
```

## 2.5 词法分析器的关键词识别流程

当词法分析器遇到一个标识符起始字符时，完整流程如下：

1. 收集连续字符：扫描所有满足标识符规则的字符（字母、数字、下划线、CJK统一表意字符），组成一个词素（lexeme）。
2. 查询 KeywordTable：以词素为键在 `KeywordTable.map` 中查询。
3. 若命中：返回对应的 `TokenKind`（如 `TokenKind::LET`）。词素被识别为关键词。
4. 若未命中：返回 `TokenKind::Ident(词素)`。词素被识别为普通标识符。

伪代码：
```rust
fn lex_identifier_or_keyword(&amp;mut self, start: SourcePosition) -&gt; Result&lt;Token, LexError&gt; {
    let mut lexeme = String::new();
    while self.current_char().map_or(false, |c| is_identifier_char(c)) {
        lexeme.push(self.advance().unwrap());
    }
    let kind = self.keyword_table.get(&amp;lexeme)
        .unwrap_or_else(|| TokenKind::Ident(lexeme.clone()));
    Ok(Token { kind, span: ..., lexeme })
}
```

## 2.6 关键词混用规则

- **任意混用**：开发者可以在同一语句、同一行、甚至同一表达式中混用三语关键词。例如 `if 年龄 &gt; 18 then` 是合法的。
- **标识符与关键词冲突处理**：若一个标识符与三语关键词的任一种形式拼写相同，词法分析器优先将其识别为关键词。因此，不能使用 `let`、`令`、`ling` 作为变量名。若确实需要，建议使用下划线或大小写变体（如 `let_`、`Let`）。编译器可发出警告提示开发者。
- **符号形式与文字形式等效**：`&amp;&amp;` 与 `且` 等效，`+` 与 `加` 等效。但为保持一致性，推荐使用符号形式（除非开发者有特殊偏好）。

## 2.7 关键词风格转换

### 2.7.1 转换算法

关键词风格转换基于完整 AST 而非 Token 流，这样可以精确保留空白、注释等格式信息。转换过程：

1. 解析源文件得到 AST。
2. 遍历 AST 的每个节点。
3. 当遇到属于关键词的 `Ident` 或 `TokenKind` 时，根据目标风格（中文/拼音/英文）替换其文本表示。
4. 对于非关键词的标识符和所有字面量，保持原样。
5. 对于注释和空白，保持原样。

### 2.7.2 命令行接口

```bash
huan fmt &lt;输入文件&gt; --style &lt;chinese|pinyin|english&gt; [--output &lt;输出文件&gt;]
huan fmt &lt;输入文件&gt; --style chinese -w    # 直接覆盖原文件
huan fmt &lt;输入文件&gt; --style english       # 输出到标准输出
```

### 2.7.3 实现接口

```rust
pub struct KeywordStyleConverter {
    table: KeywordTable,
}

impl KeywordStyleConverter {
    pub fn convert(&amp;self, program: &amp;Program, style: KeywordStyle) -&gt; String {
        let mut emitter = CodeEmitter::new(style, &amp;self.table);
        emitter.emit_program(program)
    }
}
```

## 2.8 扩展性与国际化

三语关键词系统的设计允许未来扩展更多语言支持：

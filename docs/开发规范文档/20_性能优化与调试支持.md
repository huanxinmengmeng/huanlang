# 20. 性能优化与调试支持

## 20.1 性能优化概述

幻语编译器提供多层级的性能优化机制，从高级抽象的内联展开到底层 LLVM 的激进优化，确保生成的机器码达到与手写 C/Rust 相当的性能水平。同时，完善的调试支持使开发者能够高效定位问题。

**核心能力**：
- **分级优化选项**：从快速编译的 `-O0` 到激进优化的 `-O3`、`-Os`、`-Oz`
- **链接时优化（LTO）**：跨模块内联与死代码消除
- **配置文件引导优化（PGO）**：基于运行时数据的优化
- **Post-LTO 优化**：链接后进一步优化
- **DWARF 调试信息**：支持源码级调试
- **性能剖析工具集成**：生成 perf 映射、火焰图数据
- **栈跟踪与错误报告**：panic 时的详细调用栈

## 20.2 编译器优化级别

### 20.2.1 优化级别定义

| 级别 | 命令行 | 说明 | 适用场景 |
|------|--------|------|---------|
| `0` | `-O0` | 无优化，快速编译，保留完整调试信息 | 开发调试 |
| `1` | `-O1` | 基本优化（内联、常量折叠、简单死代码消除） | 快速迭代测试 |
| `2` | `-O2` | 标准优化（循环展开、向量化、全局值编号） | 生产发布（默认） |
| `3` | `-O3` | 激进优化（更多内联、循环展开、过程间优化） | 性能关键代码 |
| `s` | `-Os` | 优化代码尺寸（类似 `-O2` 但禁用增加尺寸的优化） | 嵌入式/移动端 |
| `z` | `-Oz` | 激进尺寸优化（禁用可能增加尺寸的所有优化） | 极小固件 |

### 20.2.2 优化 Pass 详细说明

**`-O0`（无优化）**：
- 禁用所有优化 Pass
- 每个语句生成独立的代码块
- 变量均分配在栈上，便于调试器观察
- 编译速度最快

**`-O1`（基本优化）**：
- 常量折叠与传播
- 代数简化（如 `x * 1` → `x`）
- 死代码消除
- 简单内联（小函数、单次调用）
- 尾调用优化

**`-O2`（标准优化）**：
- 包含 `-O1` 所有优化
- 循环不变量外提
- 循环展开（小循环）
- 自动向量化（SIMD）
- 全局值编号（GVN）
- 指令合并
- 跳转线程化

**`-O3`（激进优化）**：
- 包含 `-O2` 所有优化
- 更激进的内联策略
- 循环展开（更大阈值）
- 过程间优化（跨函数边界）
- 预测性公共子表达式消除

**`-Os`（尺寸优化）**：
- 基于 `-O2`，但禁用可能增加代码尺寸的优化
- 优先使用更紧凑的指令序列
- 减少内联

**`-Oz`（激进尺寸优化）**：
- 基于 `-Os`
- 进一步减少代码尺寸，甚至牺牲少量性能
- 使用更小的栈帧布局

### 20.2.3 优化选项配置

在 `幻语包.toml` 中配置：

```toml
[profile.release]
opt-level = 3
lto = true
codegen-units = 1
panic = "abort"
strip = true
```

## 20.3 链接时优化（LTO）

### 20.3.1 LTO 模式

幻语通过 LLVM 支持多级 LTO：

| 模式 | 命令行标志 | 说明 |
|------|-----------|------|
| 禁用 | 默认 | 无跨模块优化 |
| Thin LTO | `--lto=thin` | 轻量级 LTO，并行执行，增量编译友好 |
| Full LTO | `--lto=full` | 完整 LTO，单个 IR 模块，优化效果最好但慢 |

### 20.3.2 LTO 配置

```toml
[profile.release]
lto = "thin"        # 或 "full"、false

# Thin LTO 特定选项
[profile.release.lto]
thin = true
jobs = 4            # 并行任务数
import-instr-limit = 100  # 导入指令限制
```

### 20.3.3 LTO 优化效果

- **内联跨模块函数**：消除函数调用开销
- **死代码消除**：移除未使用的跨模块代码
- **常量传播**：跨模块传播常量值
- **类型去虚拟化**：将特征对象调用转化为直接调用

## 20.4 配置文件引导优化（PGO）

### 20.4.1 PGO 工作流程

PGO 使用真实工作负载的运行时数据指导优化决策。

**三步流程**：

1. **编译插桩版本**：
```bash
huan build --release --pgo-generate 程序.hl
```

2. **运行训练工作负载**：
```bash
./程序 --训练数据
```
运行后生成 `default_*.profraw` 文件。

3. **编译优化版本**：
```bash
llvm-profdata merge -o 程序.profdata default_*.profraw
huan build --release --pgo-use=程序.profdata 程序.hl
```

### 20.4.2 PGO 优化内容

基于运行时数据，编译器可以：

- **优化分支预测**：将热路径代码放在一起
- **内联热函数**：更激进地内联频繁调用的函数
- **寄存器分配优化**：为热变量分配寄存器
- **函数布局优化**：将频繁调用的函数放在相近位置
- **循环展开决策**：基于实际迭代次数决定展开程度

### 20.4.3 配置示例

```toml
[profile.pgo]
generate = true                     # 生成插桩代码
use = "path/to/程序.profdata"       # 使用配置文件
sample = false                      # 是否使用采样 PGO（AutoFDO）
```

## 20.5 调试信息生成

### 20.5.1 调试信息格式

| 格式 | 平台 | 说明 |
|------|------|------|
| DWARF | Linux, macOS, 裸机 | 标准调试格式 |
| CodeView | Windows | MSVC 调试格式 |
| Source Map | WebAssembly | Wasm 调试映射 |

### 20.5.2 调试级别

```bash
huan build --debug        # 生成完整调试信息（-g）
huan build --debug=line   # 仅行号信息（-gline-tables-only）
huan build --debug=limited # 有限调试信息
```

**配置**：
```toml
[profile.dev]
debug = true                    # 或 "line", "limited"
split-debuginfo = "unpacked"    # 或 "packed", "off"
```

### 20.5.3 DWARF 信息内容

幻语编译器生成的 DWARF 信息包括：

- **源文件位置**：将机器码映射到 `.hl` 源文件的行列
- **类型信息**：幻语类型的 DWARF 表示
- **变量位置**：变量在栈或寄存器中的位置
- **内联函数信息**：内联展开的原始调用栈
- **宏信息**：幻语宏展开的调试信息

### 20.5.4 调试信息分离

为减小可执行文件体积，可将调试信息分离到独立文件：

```bash
huan build --debug --split-debuginfo=packed
```

生成的文件：
- `程序`：剥离调试信息的可执行文件
- `程序.dwp`：打包的调试信息

## 20.6 性能剖析工具集成

### 20.6.1 perf 映射文件

对于 JIT 或裸机程序，生成 `/tmp/perf-<pid>.map` 文件：

```bash
huan build --perf-map 程序.hl
```

映射文件格式：
```
地址 长度 符号名
400000 50 斐波那契
400050 30 主
```

### 20.6.2 火焰图生成

配合 Brendan Gregg 的 FlameGraph 工具：

```bash
# 录制 perf 数据
perf record -F 99 -g ./程序

# 生成火焰图
perf script | stackcollapse-perf.pl | flamegraph.pl > flame.svg
```

### 20.6.3 自定义追踪点

使用 `幻语.追踪` 模块插入用户态追踪点：

```hl
导入 幻语.追踪 为 追踪

函数 复杂计算(输入: 整数) 返回 整数
开始
    追踪.开始("复杂计算", {"输入": 输入})
    令 结果 = 实际计算(输入)
    追踪.结束("复杂计算", {"结果": 结果})
    返回 结果
结束
```

支持的追踪后端：
- **LTTng**（Linux）
- **ETW**（Windows）
- **Chrome Trace**（跨平台，生成 JSON）

### 20.6.4 硬件计数器访问

```hl
导入 幻语.性能 为 性能

函数 测量性能()
开始
    令 计数器 = 性能.新建计数器()
    计数器.开始()
    
    运行被测代码()
    
    计数器.停止()
    
    显示("周期数: {}", 计数器.周期())
    显示("指令数: {}", 计数器.指令())
    显示("缓存未命中: {}", 计数器.缓存未命中())
结束
```

## 20.7 栈跟踪与错误报告

### 20.7.1 Panic 时的栈展开

幻语的 panic 机制提供详细的调用栈信息：

```
程序 panic: 索引越界
    at 源码/列表.hl:42
    in 列表::获取(索引: 5)
    in 主函数::处理数据()
    in 主函数::主()
```

**配置 panic 行为**：

```toml
[profile.release]
panic = "abort"   # 立即终止，不展开栈（减小体积）
panic = "unwind"  # 展开栈，清理资源（默认）
```

### 20.7.2 自定义 Panic 处理器

对于裸机环境，可定义自定义 panic 处理器：

```hl
@panic_处理器
函数 自定义panic(信息: 借用 字符串, 位置: 借用 位置)
开始
    -- 写入错误日志或闪烁 LED
    写入闪存日志(信息)
    当 真 循环
        汇编!("wfi")
    结束
结束
```

### 20.7.3 栈回溯生成

在调试模式下，幻语运行时可以生成栈回溯：

```hl
导入 幻语.调试 为 调试

函数 调试辅助()
开始
    令 回溯 = 调试.捕获栈回溯()
    显示(回溯)
    -- 输出:
    -- 0: 调试辅助 at 调试工具.hl:15
    -- 1: 业务逻辑 at 主程序.hl:42
    -- 2: 主 at 主程序.hl:10
结束
```

## 20.8 错误报告增强

### 20.8.1 富文本错误输出

编译器错误支持彩色输出和上下文显示：

```
错误[E001]: 类型不匹配
  --> 源码/主程序.hl:10:15
   |
10 |     令 年龄: 整数 = "二十五"
   |               ^^^ 期望 整数，实际是 字符串
   |
帮助: 考虑使用 转为整数() 方法: "二十五".转为整数()
```

### 20.8.2 错误代码索引

所有编译错误都有唯一代码，可通过 `huan explain` 查看详细说明：

```bash
huan explain E001
```

输出：
```
E001: 类型不匹配

表达式的类型与期望的类型不一致。

示例:
  错误: 令 年龄: 整数 = "二十五"
  正确: 令 年龄: 整数 = 25
```

### 20.8.3 JSON 错误输出

用于 IDE 集成的结构化错误格式：

```bash
huan build --error-format=json 程序.hl
```

输出：

## 20.9 编译时间优化

### 20.9.1 增量编译

幻语支持增量编译，只重新编译修改过的模块：

```bash
huan build --incremental
```

**缓存机制**：
- 依赖图缓存：`target/debug/incremental/`
- 编译产物缓存：`target/debug/deps/`
- 元数据缓存：避免重复解析依赖

### 20.9.2 并行编译

利用多核加速编译：

```bash
huan build --jobs 8
```

**配置**：
```toml
[build]
jobs = 8  # 默认 CPU 核心数
```

### 20.9.3 编译缓存（sccache 集成）

支持共享编译缓存：

```bash
export SCCACHE_BUCKET=my-bucket
export SCCACHE_REGION=us-east-1
huan build  # 自动使用 sccache
```

## 20.10 代码精简（Minification）

### 20.10.1 发布构建精简

```toml
[profile.release]
strip = true            # 移除符号表
lto = true              # 链接时优化
```

```toml
[build]
# 标记哪些函数可以被安全移除
dead-code-elimination = true
# 不生成未使用函数的代码
no-unused-functions = true
```

### 20.10.3 Wasm 体积优化

```bash
huan build --target wasm32-unknown-unknown --release --wasm-opt
```

Wasm 专用优化：
- `wasm-opt -Oz`：激进尺寸优化
- `wasm-snip`：移除未使用的函数
- `wasm-gc`：垃圾回收 Wasm 模块

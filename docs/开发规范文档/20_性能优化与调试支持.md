# 20. 性能优化与调试支持

**版本**: v0.4.0
**许可证**: Apache-2.0
**更新日期**: 2026年5月
**适用对象**: 性能工程师、调试工程师

> **说明**: 本章详细描述幻语的性能优化和调试支持机制，包括优化选项、调试信息和性能分析工具。

---

## 20.1 概述

幻语编译器提供了多层级的性能优化机制，从高级抽象的内联展开到底层 LLVM 的激进优化，确保生成的机器码达到与手写 C/Rust 相当的性能水平。完善的调试支持使开发者能够高效定位问题。

**核心能力**：
- **分级优化选项**：从快速编译的 `-O0` 到激进优化的 `-O3`、`-Os`、`-Oz`。
- **链接时优化（LTO）**：跨模块内联与死代码消除，分为 Thin LTO 和 Full LTO。
- **配置文件引导优化（PGO）**：基于运行时数据的优化，提升分支预测和内联决策。
- **调试信息生成**：DWARF 调试信息，支持源码级调试和变量查看。
- **错误报告增强**：富文本彩色错误输出，JSON 格式供 IDE 集成。
- **性能剖析工具集成**：生成 perf 映射、火焰图、自定义追踪点以及硬件计数器访问。
- **编译时间优化**：增量编译、并行编译以及 sccache 集成，加速迭代。

## 20.2 编译器优化级别

优化级别通过 `-O` 选项控制，默认为 `2`。每个级别启用一组 LLVM 优化 Pass。

| 级别 | 命令行 | 说明 | 适用场景 |
|------|--------|------|---------|
| `0` | `-O0` | 无优化，快速编译，保留完整调试信息 | 开发调试 |
| `1` | `-O1` | 基本优化（内联、常量折叠、简单死代码消除） | 快速迭代测试 |
| `2` | `-O2` | 标准优化（循环展开、向量化、全局值编号） | 生产发布（默认） |
| `3` | `-O3` | 激进优化（更多内联、循环展开、过程间优化） | 性能关键代码 |
| `s` | `-Os` | 优化代码尺寸（类似 `-O2` 但禁用增加尺寸的优化） | 嵌入式/移动端 |
| `z` | `-Oz` | 激进尺寸优化（禁用可能增加尺寸的所有优化） | 极小固件 |

优化选项可在 `幻语包.toml` 中配置：
```toml
[profile.release]
opt-level = 3
lto = true
codegen-units = 1
panic = "abort"
strip = true
```

## 20.3 链接时优化（LTO）

LTO 允许编译器在链接阶段对整个程序进行跨模块优化。幻语通过 LLVM 提供 Thin LTO 和 Full LTO 两种模式。

### 20.3.1 模式选择

| 模式 | 命令行标志 | 说明 |
|------|-----------|------|
| 禁用 | 默认 | 无跨模块优化 |
| Thin LTO | `--lto=thin` | 轻量级 LTO，并行执行，增量编译友好 |
| Full LTO | `--lto=full` | 完整 LTO，单个 IR 模块，优化效果最好但慢 |

### 20.3.2 配置示例

```toml
[profile.release]
lto = "thin"        # 或 "full"、false

# Thin LTO 特定选项
[profile.release.lto]
thin = true
jobs = 4            # 并行任务数
import-instr-limit = 100  # 导入指令限制
```

LTO 优化效果包括：跨模块内联、死代码消除、常量传播和类型去虚拟化。

## 20.4 配置文件引导优化（PGO）

PGO 使用真实工作负载的运行时数据指导优化，通常可带来 10%-20% 的性能提升。

### 20.4.1 工作流程

**步骤 1：生成插桩版本**
```bash
huan build --release --pgo-generate 程序.hl
```

**步骤 2：运行训练工作负载**
```bash
./程序 --训练数据
```
运行后生成 `default_*.profraw` 文件。

**步骤 3：合并配置文件并编译优化版本**
```bash
llvm-profdata merge -o 程序.profdata default_*.profraw
huan build --release --pgo-use=程序.profdata 程序.hl
```

### 20.4.2 优化内容

基于运行时数据，编译器可以：
- 优化分支预测，将热路径放在一起。
- 更激进地内联频繁调用的函数。
- 为热变量分配寄存器。
- 调整函数布局以提升指令缓存命中率。
- 基于真实迭代次数决定循环展开程度。

### 20.4.3 配置

```toml
[profile.pgo]
generate = true                     # 生成插桩代码
use = "path/to/程序.profdata"       # 使用配置文件
sample = false                      # 使用采样 PGO（AutoFDO）
```

## 20.5 调试信息生成

### 20.5.1 调试信息格式

| 格式 | 平台 | 说明 |
|------|------|------|
| DWARF | Linux, macOS, 裸机 | 标准调试格式 |
| CodeView | Windows | MSVC 调试格式 |
| Source Map | WebAssembly | Wasm 调试映射 |

### 20.5.2 调试级别

```bash
huan build --debug                      # 完整调试信息（-g）
huan build --debug=line                 # 仅行号信息（-gline-tables-only）
huan build --debug=limited              # 有限调试信息
```

配置：
```toml
[profile.dev]
debug = true                    # 或 "line", "limited"
split-debuginfo = "unpacked"    # 或 "packed", "off"
```

### 20.5.3 DWARF 信息内容

幻语编译器生成的 DWARF 信息包括：
- 源文件位置（映射机器码到 `.hl` 源文件）。
- 幻语类型在 DWARF 中的表示。
- 变量在栈或寄存器中的位置。
- 内联函数展开的原始调用栈。
- 宏展开的调试信息。

### 20.5.4 调试信息分离

为减小可执行文件体积，可将调试信息分离到独立文件：

```bash
huan build --debug --split-debuginfo=packed
```

生成的文件：`程序`（剥离调试信息）和 `程序.dwp`（打包的调试信息）。

## 20.6 性能剖析工具集成

### 20.6.1 perf 映射文件

对于 JIT 或裸机程序，生成 `/tmp/perf-<pid>.map` 文件，将地址映射到函数名。

```bash
huan build --perf-map 程序.hl
```

映射文件格式：
```
400000 50 斐波那契
400050 30 主
```

### 20.6.2 火焰图生成

配合 Brendan Gregg 的 FlameGraph 工具：

```bash
perf record -F 99 -g ./程序
perf script | stackcollapse-perf.pl | flamegraph.pl > flame.svg
```

### 20.6.3 自定义追踪点

使用 `幻语.性能` 或 `幻语.追踪` 模块插入用户态追踪点：

```hl
导入 幻语.追踪 为 追踪

函数 复杂计算(输入: 整数) -> 整数
开始
    追踪.开始("复杂计算", {"输入": 输入})
    令 结果 = 实际计算(输入)
    追踪.结束("复杂计算", {"结果": 结果})
    返回 结果
结束
```

支持的后端：LTTng (Linux)、ETW (Windows)、Chrome Trace (跨平台 JSON)。

### 20.6.4 硬件计数器访问

```hl
导入 幻语.性能 为 性能

函数 测量性能()
开始
    令 计数器 = 性能.新建计数器()
    计数器.开始()
    运行被测代码()
    计数器.停止()
    显示("周期数: {}", 计数器.周期())
    显示("指令数: {}", 计数器.指令())
    显示("缓存未命中: {}", 计数器.缓存未命中())
结束
```

## 20.7 栈跟踪与错误报告

### 20.7.1 Panic 时的栈展开

幻语的 panic 机制提供详细的调用栈信息：

```
程序 panic: 索引越界
    at 源码/列表.hl:42
    in 列表::获取(索引: 5)
    in 主函数::处理数据()
    in 主函数::主()
```

配置 panic 行为：

```toml
[profile.release]
panic = "abort"   # 立即终止，不展开栈（减小体积）
panic = "unwind"  # 展开栈，清理资源（默认）
```

### 20.7.2 自定义 Panic 处理器

对于裸机环境，可定义自定义 panic 处理器：

```hl
@panic_处理器
函数 自定义panic(信息: 借用 字符串, 位置: 借用 位置)
开始
    写入闪存日志(信息)
    当 真 循环
        汇编!("wfi")
    结束
结束
```

### 20.7.3 栈回溯生成

在调试模式下，幻语运行时可以生成栈回溯：

```hl
导入 幻语.调试 为 调试

函数 调试辅助()
开始
    令 回溯 = 调试.捕获栈回溯()
    显示(回溯)
    -- 输出:
    -- 0: 调试辅助 at 调试工具.hl:15
    -- 1: 业务逻辑 at 主程序.hl:42
    -- 2: 主 at 主程序.hl:10
结束
```

## 20.8 错误报告增强

### 20.8.1 富文本错误输出

编译器错误支持彩色输出和上下文显示：

```
错误[E001]: 类型不匹配
  --> 源码/主程序.hl:10:15
   |
10 |     令 年龄: 整数 = "二十五"
   |               ^^^ 期望 整数，实际是 字符串
   |
帮助: 考虑使用 转为整数() 方法: "二十五".转为整数()
```

### 20.8.2 错误代码索引

所有编译错误都有唯一代码，可通过 `huan explain` 查看详细说明：

```bash
huan explain E001
```

输出：
```
E001: 类型不匹配

表达式的类型与期望的类型不一致。

示例:
  错误: 令 年龄: 整数 = "二十五"
  正确: 令 年龄: 整数 = 25
```

### 20.8.3 JSON 错误输出

用于 IDE 集成的结构化错误格式：

```bash
huan build --error-format=json 程序.hl
```

```json
{
  "message": "类型不匹配",
  "code": "E001",
  "level": "error",
  "spans": [{
    "file": "源码/主程序.hl",
    "line": 10,
    "column": 15,
    "label": "期望 整数，实际是 字符串"
  }],
  "children": [],
  "suggestions": ["考虑使用 转为整数() 方法"]
}
```

## 20.9 编译时间优化

### 20.9.1 增量编译

幻语支持增量编译，只重新编译修改过的模块：

```bash
huan build --incremental
```

缓存机制：依赖图缓存、编译产物缓存、元数据缓存。

### 20.9.2 并行编译

利用多核加速编译：

```bash
huan build --jobs 8
```

配置：
```toml
[build]
jobs = 8  # 默认 CPU 核心数
```

### 20.9.3 编译缓存（sccache 集成）

支持共享编译缓存：

```bash
export SCCACHE_BUCKET=my-bucket
export SCCACHE_REGION=us-east-1
huan build  # 自动使用 sccache
```

## 20.10 代码精简（Minification）

### 20.10.1 发布构建精简

```toml
[profile.release]
strip = true            # 移除符号表
lto = true              # 链接时优化
panic = "abort"         # 移除栈展开代码
codegen-units = 1       # 单代码生成单元，利于优化
```

### 20.10.2 死代码消除配置

```toml
[build]
dead-code-elimination = true
no-unused-functions = true
```

### 20.10.3 Wasm 体积优化

```bash
huan build --target wasm32-unknown-unknown --release --wasm-opt
```

Wasm 专用优化：`wasm-opt -Oz`（激进尺寸优化）、`wasm-snip`（移除未使用的函数）。

## 20.12 调试工具集成

### 20.12.1 GDB/LLDB 集成

幻语编译器生成兼容 GDB/LLDB 的调试信息，支持断点设置、变量查看、表达式求值、调用栈查看。

幻语专用 GDB 脚本示例：
```python
# .gdbinit
define pstr
  call huan_debug_print_string($arg0)
end
```

### 20.12.2 远程调试

对于嵌入式目标：

```bash
# 启动 GDB 服务器
openocd -f board/stm32f4discovery.cfg

# 连接调试器
huan debug --remote :3333 ./固件.elf
```

### 20.12.3 时间旅行调试

通过 `rr` 录制并回放程序执行：

```bash
rr record ./程序
rr replay
```

## 20.13 性能基准测试

### 20.13.1 微基准测试框架

```hl
导入 幻语.基准 为 基准

基准.组("字符串操作", || {
    基准.测试("连接", || {
        令 结果 = "Hello" + " " + "World"
        基准.防止优化(结果)
    })
    
    基准.测试("分割", || {
        令 结果 = "a,b,c,d,e".分割(",")
        基准.防止优化(结果)
    })
})
```

### 20.13.2 性能回归检测

```bash
# 保存基线
huan bench --save-baseline 基线

# 比较变更
huan bench --baseline 基线
```

输出：
```
基准测试结果比较:
  名称                    基线         当前         变化
  连接                    12.3 ns      12.1 ns      -1.6%
  分割                    45.6 ns      48.2 ns      +5.7% (回归)
```

## 20.14 测试用例

```rust
#[cfg(test)]
mod tests {

    }
}
```

## 本章总结

本章详细描述了幻语性能优化与调试支持的完整规范，包括：

1. **性能优化概述**：编译时优化（LLVM IR优化）和运行时优化（内联、逃逸分析）两个层次

3. **调试支持**：DWARF调试信息生成和源码级调试支持
4. **性能分析**：perf、火焰图、量化指标的性能分析工具支持
5. **错误消息格式**：人性化错误提示、错误代码系统和JSON格式输出
6. **内存分析**：Valgrind、AddressSanitizer、MemorySanitizer的集成支持
7. **性能计数器**：内置性能计数器和统计信息收集
8. **基准测试**：性能回归检测和基线比较工具
9. **测试用例**：调试信息生成和错误格式的完整测试

这些规范为幻语的调试和性能优化提供了完整的工具支持。

# 幻语编程语言 - 开发规范文档
# 16. 工具链与命令行接口

**版本**: v0.4.0
**许可证**: Apache-2.0
**更新日期**: 2026年5月
**适用对象**: 工具链开发者、命令行用户

> **说明**: 本章详细描述幻语工具链的命令行接口，包括编译、运行、调试和包管理命令。

---

## 16.1 概述

幻语工具链提供了一套完整的命令行接口（CLI），覆盖从代码编写、编译、运行、调试到包管理的全流程。所有命令均遵循一致的设计模式，支持详细的帮助信息和错误报告。

**设计原则**：
- **统一入口**：所有功能通过 `huan` 命令访问。
- **子命令结构**：功能按子命令分组，如 `huan build`、`huan run`、`huan test`。
- **丰富的选项**：支持短选项和长选项，关键选项提供默认值。
- **中文友好**：命令和选项均支持中文名称（同时兼容拼音和英文别名）。
- **环境变量集成**：可通过环境变量预设常用选项。

## 16.2 主命令接口

### 16.2.1 命令格式

```bash
huan [全局选项] <命令> [命令选项] [参数]
```

### 16.2.2 全局选项

| 短选项 | 长选项 | 中文别名 | 说明 |
|--------|--------|---------|------|
| `-h` | `--help` | `--帮助` | 显示帮助信息 |
| `-V` | `--version` | `--版本` | 显示版本信息 |
| `-v` | `--verbose` | `--详细` | 详细输出（可重复，如 `-vv`） |
| `-q` | `--quiet` | `--安静` | 减少输出，仅显示错误 |
| | `--color` | `--颜色` | 控制彩色输出：`auto`、`always`、`never` |
| | `--config` | `--配置` | 指定配置文件路径（默认 `幻语包.toml`） |

### 16.2.3 环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `HUAN_PATH` | 幻语安装路径 | `/usr/local/huan` |
| `HUAN_PACKAGE_PATH` | 包缓存路径 | `~/.huan/registry/cache` |
| `HUAN_TARGET` | 默认目标平台 | `x86_64-unknown-linux-gnu` |
| `HUAN_OPT_LEVEL` | 默认优化级别 | `2` |
| `HUAN_COLOR` | 默认颜色设置 | `auto` |
| `HUAN_JOBS` | 并行编译任务数 | `4` |

## 16.3 子命令全集

### 16.3.1 构建命令（`build` / `构建`）

编译源文件，生成可执行文件、库或目标文件。

```bash
huan build [选项] <输入文件>
```

**选项**：

| 短选项 | 长选项 | 中文别名 | 参数 | 默认值 | 说明 |
|--------|--------|---------|------|--------|------|
| `-o` | `--output` | `--输出` | `<文件>` | 基于输入文件名 | 输出文件路径 |
| `-O` | `--opt-level` | `--优化` | `0-3, s, z` | `2` | 优化级别 |
| | `--target` | `--目标` | `<三元组>` | `host` | 目标平台三元组 |
| | `--ownership` | `--所有权` | - | `false` | 启用所有权检查 |
| `-g` | `--debug` | `--调试符号` | - | `false` | 生成调试信息 |
| | `--release` | `--发布` | - | `false` | 发布模式（等同 `-O 3 --no-debug`） |
| | `--pic` | `--位置无关` | - | `false` | 生成位置无关代码 |
| | `--stack-protector` | `--栈保护` | - | `false` | 启用栈保护 |
| | `--target-features` | `--目标特性` | `<特性>` | - | 目标 CPU 特性，如 `+avx2` |
| | `--jobs` | `--并行` | `<数量>` | CPU 核心数 | 并行编译任务数 |
| | `--emit` | `--生成` | `llvm-ir,asm,obj,link` | `link` | 生成的文件类型 |
| | `--extern` | `--外部库` | `<路径>` | - | 添加库搜索路径 |
| `-L` | `--library-path` | `--库路径` | `<路径>` | - | 添加库搜索路径 |
| `-l` | `--link` | `--链接` | `<库名>` | - | 链接外部库 |

**示例**：

```bash
# 基本构建
huan build 程序.hl

# 指定输出和优化级别
huan build 程序.hl -o 程序 -O 3

# 构建为库
huan build 库.hl --emit lib

# 交叉编译到 ARM64 Linux
huan build 程序.hl --target aarch64-unknown-linux-gnu

# 发布构建
huan build 程序.hl --release
```

### 16.3.2 运行命令（`run` / `运行`）

编译并运行源文件。如果已编译且未修改，则直接运行。

```bash
huan run [选项] <输入文件> [--] [程序参数]
```

选项继承 `build` 的所有选项，外加：

| 短选项 | 长选项 | 中文别名 | 说明 |
|--------|--------|---------|------|
| | `--no-build` | `--不构建` | 跳过编译，直接运行已有可执行文件 |
| | `--args` | `--参数` | 传递给程序的参数 |

**示例**：

```bash
huan run 程序.hl
huan run 程序.hl -- --help
huan run --no-build ./程序
```

### 16.3.3 交互命令（`repl` / `交互`）

启动交互式环境（REPL），逐行执行幻语代码。

```bash
huan repl [选项]
```

| 短选项 | 长选项 | 中文别名 | 说明 |
|--------|--------|---------|------|
| | `--no-history` | `--无历史` | 不保存命令历史 |
| | `--eval` | `--执行` | 执行指定表达式后退出 |

**REPL 特殊命令**：

| 命令 | 说明 |
|------|------|
| `.help` / `.帮助` | 显示帮助 |
| `.quit` / `.退出` | 退出 REPL |
| `.clear` / `.清屏` | 清屏 |
| `.show` / `.显示` | 显示上次结果 |
| `.load` / `.加载` | 加载并执行文件 |
| `.save` / `.保存` | 保存会话历史到文件 |

### 16.3.4 检查命令（`check` / `检查`）

仅执行语法和类型检查，不生成代码。

```bash
huan check [选项] <输入文件>
```

选项继承 `build` 的编译相关选项。

**示例**：

```bash
huan check 程序.hl
huan check -v 程序.hl
```

### 16.3.5 格式化命令（`fmt` / `格式化`）

格式化源代码，并可转换关键词风格。

```bash
huan fmt [选项] <输入文件>
```

| 短选项 | 长选项 | 中文别名 | 参数 | 默认值 | 说明 |
|--------|--------|---------|------|--------|------|
| | `--style` | `--风格` | `chinese, pinyin, english` | 保持原风格 | 输出关键词风格 |
| `-w` | `--write` | `--写入` | - | `false` | 直接写入文件（而非输出到 stdout） |
| | `--check` | `--检查` | - | `false` | 检查是否已格式化，不修改文件 |
| | `--tab-size` | `--缩进宽度` | `<数值>` | `4` | 缩进空格数 |
| | `--use-tabs` | `--使用制表符` | - | `false` | 使用制表符而非空格 |

**示例**：

```bash
huan fmt 程序.hl
huan fmt 程序.hl --style chinese -w
huan fmt 程序.hl --check
```

### 16.3.6 编辑命令（`edit` / `编辑`）

```bash
huan edit [文件] [选项]
```

| 短选项 | 长选项 | 中文别名 | 说明 |

| | `--line` | `--行` | 跳转到指定行 |

| | `--config` | `--配置` | 指定编辑器配置文件 |

**示例**：

```bash
huan edit 程序.hl
```

### 16.3.7 转换命令（`transpile` / `转换`）

将幻语代码转换为其他语言，或将其它语言转换为幻语。

```bash
huan transpile [选项] <输入文件>
```

| 短选项 | 长选项 | 中文别名 | 参数 | 说明 |
|--------|--------|---------|------|------|
| | `--from` | `--源语言` | `c,cpp,rust,python,java,go,js,ts,huan` | 源语言（默认根据扩展名推断） |
| | `--to` | `--目标` | `c,cpp,rust,python,java,go,js,ts,kotlin,swift,dart,carbon,zig,huan` | 目标语言 |
| `-o` | `--output` | `--输出` | `<文件>` | 输出文件路径 |
| | `--preserve-comments` | `--保留注释` | - | 保留注释 |
| | `--optimize` | `--优化` | - | 应用转译优化 |

**示例**：

```bash
huan transpile 程序.hl --to rust -o 程序.rs
huan transpile 脚本.py --from python --to huan -o 脚本.hl
```

### 16.3.8 导入库命令（`import-lib` / `导入库`）

导入外部库并生成幻语绑定或转换为幻语原生库。

```bash
huan import-lib <库名> [选项]
```

| 短选项 | 长选项 | 中文别名 | 参数 | 默认值 | 说明 |
|--------|--------|---------|------|--------|------|
| | `--lang` | `--语言` | `c,python,java,rust,js` | - | 源库语言（必需） |
| | `--level` | `--级别` | `dynamic,static,full` | `dynamic` | 绑定级别 |
| `-o` | `--output` | `--输出` | `<目录>` | `./huan_libs/` | 输出目录 |
| | `--header` | `--头文件` | `<路径>` | - | C 头文件路径（C 库必需） |
| | `--module` | `--模块名` | `<名称>` | 库名 | Python 模块名 |
| | `--version` | `--版本` | `<版本>` | - | 指定库版本 |

**示例**：

```bash
huan import-lib ssl --lang c --header /usr/include/openssl/ssl.h
huan import-lib numpy --lang python --level dynamic
huan import-lib requests --lang python --level full
```

## 16.4 配置文件规范（`幻语包.toml`）

### 16.4.1 完整配置结构

```toml
[package]
name = "项目名称"
version = "0.1.0"
edition = "1.2"
authors = ["作者 <邮箱>"]
description = "项目描述"
license = "MIT"
repository = "https://.../"
keyword_style = "chinese"

[dependencies]
网络 = "0.3"
序列化 = { version = "1.0", optional = true }

[dev-dependencies]
测试工具 = "0.5"

[build-dependencies]
构建辅助 = "0.1"

[extern]
c_libs = ["ssl"]
python_modules = [{ name = "numpy", version = "1.24" }]
asm_files = ["启动代码.has m"]

[lib]
name = "库名称"
path = "源码/库.hl"
crate-type = ["staticlib", "cdylib"]

[[bin]]
name = "可执行文件名称"
path = "源码/主程序.hl"

[build]
opt-level = 2
target = "host"
ownership = false
debug = false

[profile.dev]
opt-level = 0
debug = true

[profile.release]
opt-level = 3
debug = false
lto = true
strip = true
```

### 16.4.2 配置字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `package.name` | 字符串 | 包名称（必需） |

## 16.5 构建流程

### 16.5.1 编译阶段

1. **解析配置文件**：读取 `幻语包.toml`，合并命令行选项，确定目标平台和优化级别。
2. **依赖解析**：下载缺失的依赖，解析版本约束，构建依赖图。
3. **编译**：词法分析 → 语法分析（AST） → 语义分析（类型标注） → MLIR（`huan` 方言） → MLIR 降级 → LLVM IR → 目标代码。
4. **链接**：链接目标文件和外部库，生成最终可执行文件/库。

### 16.5.2 增量编译

通过文件修改时间检测、依赖图缓存、编译产物缓存和并行编译实现增量编译，加速重复构建。

### 16.5.3 交叉编译

通过指定 `--target` 选项进行交叉编译，编译器会自行选择合适的链接器和系统库路径。

## 16.6 退出码

| 退出码 | 含义 |
|--------|------|
| `0` | 成功 |
| `1` | 一般错误（编译失败、运行时错误） |
| `2` | 命令行参数错误 |
| `3` | 配置文件错误 |
| `101` | 内部编译器错误（ICE） |
| `130` | 被用户中断（Ctrl+C） |

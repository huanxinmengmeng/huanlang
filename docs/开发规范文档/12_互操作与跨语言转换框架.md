## 12.1 概述

幻语的互操作与跨语言转换框架是其全栈生态的核心支柱。通过三层架构，幻语能够直接调用现有语言库、自动生成多语言绑定、以及双向转译代码。这使得开发者可以：

- **复用现有生态**：无需重写已有的 C、Python、Java、JavaScript 等库。
- **渐进式迁移**：将现有项目逐步迁移到幻语，或从幻语导出功能到其他语言。
- **跨语言协作**：在同一个项目中混合使用不同语言的组件。
- **AI 友好**：统一的中间表示便于 AI 模型理解和生成代码。

## 12.2 第 1 层：FFI 接口层

### 12.2.1 外部块语法

幻语通过 `外部` 块声明外部语言的函数和变量。语法为：

```
外部 "<语言标识>" {
    { 外部函数声明 | 外部变量声明 | 导入语句 }
}
```

支持的语言标识包括：`"C"`、`"Python"`、`"Java"`、`"JavaScript"`、`"Rust"`、`"asm"` 等。

### 12.2.2 C FFI 调用

C FFI 是幻语与外部库交互的基础，所有兼容 C ABI 的语言均可通过此方式调用。

示例：
```hl
外部 "C" {
    函数 malloc(大小: 整数) -> 指针<无符号8>
    函数 free(指针: 指针<无符号8>)
    函数 printf(格式: 字符串, ...) -> 整数
    定 errno: 整数
}

函数 演示()
开始
    令 缓冲区 = malloc(1024)
    若 缓冲区 != 空 则
        printf("分配了 1024 字节\n")
        free(缓冲区)
    结束
结束
```

**编译器处理**：
- 解析外部块，将函数签名记录到符号表。
- 生成 LLVM IR 时，使用 C 调用约定（`ccc`）并保持名称无修饰。
- 链接时自动添加 `-l` 参数（通过包配置的 `[extern]` 节）。

### 12.2.3 Python FFI 调用

通过嵌入 CPython 解释器或生成 Python C API 包装代码，实现与 Python 模块的互操作。

```hl
外部 "Python" 导入 "numpy" 为 np
外部 "Python" 导入 "requests" 为 请求

函数 分析数据()
开始
    令 数组 = np.数组([1, 2, 3, 4, 5])
    令 均值 = np.均值(数组)
    显示("平均值: ", 均值)

    令 响应 = 请求.获取("https://api.example.com/data")
    显示("状态码: ", 响应.状态码)
结束
```

实现架构：
```
幻语 Python FFI 实现
  1. 解析 "外部 Python 导入 ..." 语句
  2. 在运行时初始化 CPython 解释器（若未初始化）
  3. 对于每个 Python 函数调用：
     - 将幻语参数转换为 PyObject*
     - 调用 PyObject_CallObject
     - 将返回值转换回幻语类型
  4. 编译时也可选择将 Python 调用转译为等效幻语代码
```

### 12.2.4 Java/Kotlin FFI 调用（通过 JNI）

```hl
外部 "Java" 导入 "java.util.HashMap" 为 哈希映射
外部 "Java" 导入 "java.lang.System" 为 系统

函数 演示()
开始
    令 映射 = 哈希映射.新建()
    映射.放置("键", "值")
    系统.输出.打印行("Hello from 幻语")
结束
```

实现方式：生成 JNI 包装代码，将幻语函数编译为共享库，由 Java 端加载调用。

### 12.2.5 JavaScript/TypeScript FFI（Wasm 环境）

```hl
外部 "JavaScript" {
    函数 控制台_日志(消息: 字符串)
    函数 获取元素(标识: 字符串) -> 指针<单元>
    函数 设置超时(回调: 函数(), 延迟: 整数)
}

控制台_日志("Hello from 幻语")
设置超时(|| { 控制台_日志("超时触发") }, 1000)
```

编译器在 Wasm 编译时生成对应的导入声明，并在 JavaScript 端提供具体实现。

### 12.2.6 外部汇编调用

```hl
外部 "asm" {
    函数 快速内存复制(目标: 指针<单元>, 源: 指针<单元>, 长度: 整数)
    函数 读取栈指针() -> 指针<单元>

令 栈顶 = 读取栈指针()
```

## 12.3 第 2 层：统一绑定生成层

### 12.3.1 导出接口定义

使用 `@导出` 属性标注需要对外暴露的 API，编译器生成 C ABI 包装层和语言绑定代码。

```hl
@导出("calculator")
模块 计算器
    @导出
    结构 点
        字段 横坐标: 浮点64
        字段 纵坐标: 浮点64
    结束

    @导出
    函数 距离(甲: 点, 乙: 点) -> 浮点64
    开始
        令 横向差 = 甲.横坐标 - 乙.横坐标
        令 纵向差 = 甲.纵坐标 - 乙.纵坐标
        返回 开平方(横向差*横向差 + 纵向差*纵向差)
    结束
结束
```

### 12.3.2 生成的多语言绑定示例

**Kotlin**：
```kotlin
data class Point(val x: Double, val y: Double)
object Calculator {
    external fun distance(a: Point, b: Point): Double
    init { System.loadLibrary("calculator") }
}
```

**Swift**：
```swift
public struct Point { public var x: Double; public var y: Double }
public func calculator_distance(_ a: Point, _ b: Point) -> Double
```

**Python**：
```python
from ctypes import CDLL, c_double, Structure
class Point(Structure):
    _fields_ = [("x", c_double), ("y", c_double)]
_lib = CDLL("./libcalculator.so")
def distance(a, b): return _lib.calculator_distance(a, b)
```

## 12.4 第 3 层：代码转译层

### 12.4.1 转译器架构

幻语采用统一的中间表示（UIR）实现双向转译：

```
源语言 AST → 幻语 UIR → 目标语言 AST → 目标语言源码
```

支持的语言包括：C, C++, Rust, Python, Java, Go, JavaScript, TypeScript, Kotlin, Swift, Dart, Carbon, Zig。

### 12.4.2 转译器接口

```rust
pub trait Transpiler {
    fn transpile(&self, program: &Program, target: TargetLanguage) -> Result<String, TranspileError>;
    fn import(&self, source: &str, source_lang: SourceLanguage) -> Result<Program, TranspileError>;
}
```

### 12.4.3 转译示例

幻语 → Rust：
```
函数 阶乘(n: 整数) -> 整数
    若 n <= 1 则 返回 1
    结束
    返回 n * 阶乘(n-1)
结束

// 转换为
fn factorial(n: i64) -> i64 {
    if n <= 1 { return 1; }
    n * factorial(n - 1)
}
```

### 12.4.4 转译实现核心

```rust
impl HuanTranspiler {
    fn transpile_function(&self, func: &Function) -> String {
        match self.target {
            TargetLanguage::Rust => {
                format!("fn {}({}) -> {} {{\n{}\n}}\n",
                    func.name.name,
                    self.format_params_rust(&func.params),
                    self.format_type_rust(&func.return_type),
                    func.body.iter().map(|s| self.transpile_stmt_rust(s)).collect::<String>()
                )
            }
            // 其他语言类似
        }
    }
}
```

## 12.5 库导入与转换优化

提供三级策略，从动态绑定到完全转译：

| 级别 | 模式 | 说明 |
|------|------|------|
| 1 | 动态绑定 | 运行时调用外部库 |
| 2 | 静态绑定 | 编译时生成 FFI 包装 |
| 3 | 完全转译 | 将库源码转换为幻语并编译优化 |

命令示例：
```bash
huan 导入库 numpy --语言 python                     # 动态
huan 导入库 numpy --语言 python --绑定风格 静态      # 静态
huan 导入库 numpy --语言 python --绑定风格 完全转译  # 完全转译为幻语库
```

## 12.6 命令行接口

```bash
# 跨语言转译
huan 转换 程序.hl --目标 rust -o 程序.rs

# 导入外部库
huan 导入库 <库名> --语言 <c|python|java|rust> [--绑定风格 动态|静态|完全] [--头文件 <路径>]

# 生成多语言绑定
huan 生成绑定 程序.hl --目标 kotlin,swift,python
```

## 12.8 测试用例

```rust
#[test]
fn test_transpile_to_rust() {

    let program = parse(source).unwrap();
```

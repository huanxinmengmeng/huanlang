# 幻语编程语言 - 开发规范文档
# 19. 包管理器与注册表规范

**版本**: v0.4.0
**许可证**: Apache-2.0
**更新日期**: 2026年5月
**适用对象**: 包开发者、库使用者

> **说明**: 本章详细描述幻语包管理器的规范，包括包格式、依赖管理、版本控制和发布流程。

---

## 19.1 概述

幻语包管理器（`huan package`）是工具链的核心组件之一，负责管理幻语项目的依赖、版本、构建和发布。它借鉴了现代包管理器（如 Cargo、npm）的最佳实践，同时针对幻语的多语言互操作特性进行了扩展。

核心功能：
- **项目初始化**：快速创建符合规范的项目骨架
- **依赖管理**：声明式依赖配置，自动解析版本约束，锁定依赖版本
- **构建与运行**：统一的项目构建和运行入口
- **包发布**：将包发布到中央注册表，供他人使用
- **包安装**：从注册表安装二进制工具或库
- **工作区支持**：管理多个相互依赖的本地包
- **外部库集成**：管理 C、Python、Rust 等外部库的绑定

## 19.2 包结构规范

### 19.2.1 包目录布局

```
包名/
├── 幻语包.toml           # 包描述文件（必需）
├── 源码/                 # 源代码目录
│   ├── 库.hl             # 库入口（可选）
│   ├── 主程序.hl         # 可执行文件入口（可选）
│   └── 模块1/            # 子模块目录
│       └── 模块.hl
├── 示例/                 # 示例代码
├── 测试/                 # 测试代码
├── 基准/                 # 基准测试代码
├── 文档/                 # 文档
├── 资源/                 # 非代码资源（图片、配置等）
├── 外部/                 # 外部库绑定声明
├── 构建脚本.hl           # 构建脚本（可选）
├── LICENSE               # 许可证文件
└── README.md             # 自述文件
```

### 19.2.2 包命名规范

- 包名只能包含小写字母、数字、连字符（`-`）和下划线（`_`）
- 包名必须以字母开头
- 包名长度限制：3-64 字符
- 推荐使用中文拼音或英文描述性名称
- 包名在注册表中唯一

```toml
[workspace]
```

## 19.4 依赖版本约束

### 19.4.1 版本约束语法

| 约束 | 示例 | 说明 |
|:------|:------|:-----|
| 精确版本 | `"1.2.3"` | 只匹配 `1.2.3` |
| 兼容版本（默认） | `"^1.2.3"` | `>=1.2.3, <2.0.0` |
| 波浪号版本 | `"~1.2.3"` | `>=1.2.3, <1.3.0` |
| 通配符 | `"1.2.*"`、`"1.*"` | 匹配任意补丁/次版本 |
| 大于/小于 | `">=1.2.0, <2.0.0"` | 自定义范围 |
| 多范围 | `">=1.0.0, <1.5.0 \|\| >=2.0.0"` | 或关系 |

### 19.4.2 语义化版本规则

幻语采用语义化版本 2.0.0：

- **主版本号（Major）**：不兼容的 API 变更
- **次版本号（Minor）**：向后兼容的功能新增
- **修订号（Patch）**：向后兼容的问题修复

**预发布版本**：`1.0.0-alpha`、`1.0.0-beta.1`、`1.0.0-rc.2`

**构建元数据**：`1.0.0+20260101`

## 19.5 锁定文件（`幻语锁.toml`）

### 19.5.1 锁定文件格式

锁定文件精确记录依赖解析结果，确保可重现构建。

```toml
# 此文件由幻语自动生成，请勿手动编辑
version = 1

[[package]]
name = "网络"
version = "0.3.0"
source = "registry+https://registry.huanlang.org"
dependencies = ["序列化 1.0.0"]

[[package]]
name = "序列化"
version = "1.0.1"
source = "registry+https://registry.huanlang.org"
checksum = "sha256:abc123..."

[[package]]
name = "本地库"
version = "0.1.0"
source = "path+../本地包"

[metadata]
"序列化 1.0.1" = { features = ["json"] }
```

### 19.5.2 锁定文件操作

- **生成/更新**：`huan package update` 或首次 `huan build`
- **使用**：存在 `幻语锁.toml` 时，使用其中记录的精确版本
- **提交**：建议将锁定文件提交到版本控制，确保团队环境一致

## 19.6 包管理器命令接口

### 19.6.1 命令总览

```bash
huan package <子命令> [选项] [参数]
```

### 19.6.2 子命令详解

#### `new` / `新建` - 创建新包

```bash
huan package new <包名> [选项]
```

| 选项 | 说明 |
|------|------|
| `--lib` | 创建库项目（默认） |
| `--bin` | 创建可执行项目 |
| `--vcs <git\|none>` | 初始化版本控制（默认 git） |
| `--edition <版本>` | 指定幻语版本 |

生成的标准目录结构：

```
包名/
├── 幻语包.toml
├── 源码/
│   └── 库.hl 或 主程序.hl
├── .gitignore
└── README.md
```

#### `build` / `构建` - 构建当前包

```bash
huan package build [选项]
```

选项继承 `huan build` 的所有选项。

#### `run` / `运行` - 运行当前包

```bash
huan package run [选项] [-- 程序参数]
```

#### `test` / `测试` - 运行测试

```bash
huan package test [选项] [测试名称模式]
```

#### `doc` / `文档` - 生成文档

```bash
huan package doc [选项]
```

#### `add` / `添加` - 添加依赖

```bash
huan package add <依赖名>[@版本] [选项]
```

| 选项 | 说明 |
|------|------|
| `--dev` | 添加为开发依赖 |
| `--build` | 添加为构建依赖 |
| `--optional` | 添加为可选依赖 |
| `--path <路径>` | 添加本地路径依赖 |
| `--git <URL>` | 添加 Git 依赖 |
| `--branch <分支>` | 指定 Git 分支 |
| `--tag <标签>` | 指定 Git 标签 |
| `--rev <提交>` | 指定 Git 提交 |
| `--features <功能>` | 启用特定功能 |

**示例**：

```bash
# 添加注册表依赖
huan package add 网络

# 添加指定版本
huan package add 序列化@1.0

# 添加开发依赖
huan package add 测试工具 --dev

# 添加本地依赖
huan package add 本地库 --path ../本地库

# 添加 Git 依赖
huan package add mylib --git https://github.com/user/mylib --branch main
```

#### `remove` / `移除` - 移除依赖

```bash
huan package remove <依赖名> [选项]
```

| 选项 | 说明 |
|------|------|
| `--dev` | 从开发依赖中移除 |
| `--build` | 从构建依赖中移除 |

#### `update` / `更新` - 更新依赖

```bash
huan package update [依赖名] [选项]
```

| 选项 | 说明 |
|------|------|
| `--precise <版本>` | 更新到精确版本 |
| `--breaking` | 允许更新主版本（破坏性更新） |
| `--dry-run` | 仅显示将要更新的内容 |

#### `publish` / `发布` - 发布包

```bash
huan package publish [选项]
```

| 选项 | 说明 |
|------|------|
| `--token <令牌>` | 认证令牌 |
| `--registry <URL>` | 指定注册表 URL |
| `--dry-run` | 模拟发布，不实际推送 |
| `--allow-dirty` | 允许未提交的更改 |
| `--no-verify` | 跳过构建验证 |

发布流程：
1. 验证包名和版本未被占用
2. 检查工作区是否干净（无未提交更改）
3. 运行 `huan package test` 确保测试通过
4. 打包源码为 `.hlp` 文件
5. 上传到注册表

#### `install` / `安装` - 安装二进制包

```bash
huan package install <包名>[@版本] [选项]
```

| 选项 | 说明 |
|------|------|
| `--bin <名称>` | 只安装指定的可执行文件 |
| `--root <目录>` | 安装根目录（默认 `~/.huan/bin`） |
| `--force` | 强制重新安装 |

#### `search` / `搜索` - 搜索包

```bash
huan package search <关键词> [选项]
```

| 选项 | 说明 |
|------|------|
| `--limit <数量>` | 返回结果数量限制 |
| `--category <分类>` | 按分类筛选 |

#### `info` / `信息` - 查看包信息

```bash
huan package info <包名>[@版本]
```

输出示例：



#### `vendor` / `本地化` - 将依赖复制到本地


#### `outdated` / `过时` - 检查过时依赖

## 19.7 注册表规范

### 19.7.1 注册表 API 端点

幻语中央注册表提供以下 REST API 端点（基础 URL: `https://registry.huanlang.org/api/v1`）：

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/crates` | 列出所有包（分页） |
| `GET` | `/crates/{name}` | 获取包信息 |
| `GET` | `/crates/{name}/{version}` | 获取特定版本信息 |
| `GET` | `/crates/{name}/{version}/download` | 下载包 |
| `PUT` | `/crates/new` | 发布新包 |
| `DELETE` | `/crates/{name}/{version}` | 撤回版本（需要认证） |
| `GET` | `/search` | 搜索包 |
| `GET` | `/categories` | 获取分类列表 |
| `GET` | `/me/crates` | 获取当前用户拥有的包（需要认证） |
| `POST` | `/login` | 获取认证令牌 |

### 19.7.3 认证机制

注册表使用 Bearer Token 进行认证：

- 通过 `POST /api/v1/login` 使用用户名/密码或 GitHub OAuth 获取临时令牌
- 令牌有效期 30 天
- 环境变量 `HUAN_TOKEN` 或 `--token` 参数传递令牌
- 令牌存储在 `~/.huan/credentials.toml`

### 19.7.4 包名抢注与命名空间

- 包名遵循先到先得原则
- 未来版本计划支持命名空间（如 `@username/package`）
- 活跃维护者可申请接管已废弃的包

### 19.8.2 索引更新机制

- 注册表索引是一个 Git 仓库，包含所有包的元数据

- 索引采用分层目录结构减少单目录文件数：`网/络/网络`

## 19.9 依赖解析算法

### 19.9.1 PubGrub 算法

幻语使用 PubGrub 算法进行依赖解析，该算法被 Cargo、Dart Pub 等广泛采用。

核心思想：
1. 将依赖约束转化为不兼容性（Incompatibility）
2. 通过冲突驱动的回溯搜索找到满足所有约束的版本组合
3. 遇到冲突时记录冲突原因，指导后续搜索

```rust
pub fn resolve(root: &Package) -> Result<Resolution, ResolveError> {
    let mut solver = PubGrubSolver::new(root);
    for dep in root.dependencies() {
        solver.add_dependency(root.id(), dep);
    }
    while let Some(next) = solver.next_package() {
        let version = solver.select_version(next)?;
        solver.add_version(next, version);
        for dep in version.dependencies() {
            solver.add_dependency(version.id(), dep);
        }
    }
    solver.into_resolution()
}
```

### 19.9.2 冲突处理与错误报告

当解析失败时，生成可读的错误信息：

```
错误: 依赖解析失败

网络 v0.3.0
└── 序列化 v1.0.0

加密 v1.2.0
└── 序列化 v2.0.0

冲突: 序列化 无法同时满足 v1.0.0 和 v2.0.0
建议:
  - 升级 网络 到支持序列化 v2.0.0 的版本
  - 降级 加密 到依赖序列化 v1.0.0 的版本
```

```toml
网络.workspace = true
```

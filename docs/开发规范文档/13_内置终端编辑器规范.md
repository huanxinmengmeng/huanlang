## 13.1 编辑器概述

幻语内置一个名为“幻语编辑”（`huan edit`）的终端文本编辑器。它提供了完整的代码编辑体验，包括 Vim 风格的模态编辑、实时语法高亮、内置 LSP 客户端、多光标编辑等功能。编辑器完全运行于终端中，无需图形界面，采用 Rust 生态的 Ratatui 和 Tree‑sitter 作为核心依赖。

**核心特性**：
- **Vim 风格的模态编辑**：支持 Normal、Insert、Visual 三种基本模式，以及命令行模式。
- **实时语法高亮**：基于 Tree‑sitter 的增量解析器，对幻语源代码进行词法高亮，并统一处理中文、拼音、英文三语关键词的着色。
- **内置 LSP 客户端**：直接连接 `huan serve` 提供的语言服务器，获取补全、诊断、跳转定义、悬停提示等智能功能。
- **多光标编辑**：支持在文档中添加多个光标，批量编辑相同文本。
- **中文输入法优化**：自动处理输入法组合事件，消除输入卡顿。
- **文件树与导航**：侧边栏文件浏览器，快速切换工作区内的文件。
- **高度可配置**：支持自定义主题、按键映射、缩进风格等。
- **剪贴板集成**：通过终端或外部工具访问系统剪贴板。
- **命令面板**：通过快捷键呼出命令列表，支持模糊搜索和执行。

## 13.2 启动与命令行接口

```bash
huan edit [文件] [选项]
```

**选项**：

| 选项 | 说明 |
|------|------|
| `--readonly` | 以只读模式打开文件 |
| `--line <行号>` | 打开文件并跳转到指定行 |
| `--col <列号>` | 与 `--line` 配合，跳转到指定列 |
| `--config <文件>` | 使用指定的配置文件（默认 `~/.huan/config.toml`） |
| `--no-plugins` | 禁用所有插件和 LSP 集成 |

## 13.3 编辑器核心数据结构

### 13.3.1 主编辑器结构

```rust
pub struct Editor {
    pub mode: Mode,
    pub buffer: Rope,                     // 文本缓冲区（高效编辑）
    pub cursors: CursorSet,              // 多光标集合
    pub highlighter: SyntaxHighlighter,  // Tree‑sitter 高亮器
    pub lsp_client: Option<LspClient>,   // LSP 集成（可关闭）
    pub config: EditorConfig,            // 用户配置
    pub renderer: TerminalRenderer,      // ANSI 渲染器
    pub file_tree: FileTree,             // 侧边文件树
    pub status_bar: StatusBar,           // 状态栏
    pub command_palette: CommandPalette, // 命令面板
    pub history: EditHistory,            // 撤销/重做历史
    pub clipboard: Clipboard,            // 剪贴板接口
    pub current_file: Option<PathBuf>,   // 当前打开文件
    pub modified: bool,                  // 是否有未保存的修改
    pub running: bool,                   // 主循环控制
}
```

### 13.3.2 编辑模式

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Mode {
    Normal,     // 普通模式（移动、命令）
    Insert,     // 插入模式（输入文本）
    Visual,     // 可视模式（字符选择）
    VisualLine, // 行可视模式
    Command,    // 命令行模式（输入 Ex 命令）
    Palette,    // 命令面板模式
}
```

### 13.3.3 文本缓冲区（Rope）

使用 `Rope` 数据结构实现高效的插入、删除和撤销操作。提供的方法包括：
- `len()`, `line_count()`, `line(index)`, `offset_of_position(line, col)`, `position_of_offset(offset)`
- `insert(offset, text)`, `remove(start, end)`, `replace(start, end, text)`, `slice(start, end)`

### 13.3.4 光标集合（多光标）

支持多个光标同时编辑，提供添加、移除、排序、去重等方法。

### 13.4.1 按键定义

```rust
pub enum Key {

```

### 13.4.2 默认按键映射（Vim 风格）

**Normal 模式**：

| 按键 | 动作 | 说明 |
|------|------|------|
| `h/j/k/l` | 光标移动 | 左/下/上/右 |
| `w/b/e` | 词移动 | 下一词首/上一词首/词尾 |
| `0/^/$` | 行移动 | 行首/首个非空字符/行尾 |
| `gg/G` | 文件移动 | 文件首/文件尾 |
| `i/a/o/O` | 进入插入模式 | 光标前/光标后/下行/上行 |
| `v/V/Ctrl+v` | 进入可视模式 | 字符/行/块选择 |
| `x/dd/D` | 删除 | 字符/行/至行尾 |
| `yy/Y` | 复制 | 行/至行尾 |
| `p/P` | 粘贴 | 光标后/光标前 |
| `u/Ctrl+r` | 撤销/重做 | |
| `//?` | 搜索 | 正向/反向 |
| `n/N` | 跳转匹配 | 下一/上一匹配 |
| `:` | 进入命令模式 | |
| `Ctrl+n` | 添加光标下一匹配 | 多光标 |
| `gd/K` | LSP 跳转/悬停 | |

**Insert 模式**：`Escape` / `Ctrl+[` 回到 Normal。`Ctrl+h` 删除前字符，`Ctrl+w` 删除前词。

**Visual 模式**：`y` 复制选中，`d` 删除选中，`Escape` 回到 Normal。

**Command 模式**：`:w` 保存，`:q` 退出，`:wq` 保存退出，`:e <文件>` 打开文件，`:sp/:vsp` 分割窗口。

## 13.5 语法高亮器

### 13.5.1 基于 Tree‑sitter 的引擎

```rust
pub struct SyntaxHighlighter {
    parser: tree_sitter::Parser,
    language: tree_sitter::Language,
    tree: Option<tree_sitter::Tree>,
    theme: Theme,
    cache: HighlightCache,
}
```

- `update(rope, edits)`：增量更新语法树。
- `highlight(rope, start_line, end_line) -> Vec<HighlightSpan>`：计算指定区域的高亮信息。
- 高亮规则由 `tree-sitter-huan` 语法定义，关键词节点统一着色，中文/拼音/英文关键词获得相同颜色。

## 13.8 配置文件格式

默认存储于 `~/.huan/config.toml`。

```toml
[theme]
name = "幻语深色"
[theme.colors]
background = "#1E1E1E"
keyword = "#569CD6"
# ...

[editor]
tab_size = 4
expand_tab = true
auto_indent = true
line_numbers = "relative"
rulers = [80, 120]

[keymap.normal]
"<C-p>" = "命令面板"
"gd" = "跳转定义"

[lsp]
server_command = "huan serve"
auto_start = true
```

## 13.9 扩展功能

**文件树**：递归展开/折叠目录，高亮当前文件，支持键盘导航打开文件。

**命令面板**：`Ctrl+p` 呼出，模糊搜索内置命令（保存、打开文件、切换主题等），支持自定义命令扩展。

## 13.10 启动流程与系统集成

1. `huan edit <file>` 被调用。
2. 解析命令行参数，加载配置文件。
3. 初始化终端（进入 raw 模式，启用鼠标支持）。
4. 创建 `Editor` 实例，打开指定文件，初始化语法高亮。
5. 若配置启用 LSP，则启动 `huan serve` 进程并建立连接。
6. 进入主循环：读取输入 → 处理按键 → 更新缓冲区 → 重绘屏幕。
7. 退出时恢复终端设置，清理 LSP 进程。

## 13.11 退出与清理

编辑器在退出前自动执行：保存未保存文件（可配置提示）、关闭 LSP 客户端、恢复终端模式、清屏并释放所有资源。

## 13.12 实现要求

所有编辑器核心组件均以 Rust 实现，利用 `crossterm` 进行终端控制，`ropey` 作为文本缓冲区，`tree-sitter` 和 `tree-sitter-huan` 提供语法解析。LSP 客户端基于 `tokio` 异步运行时。编辑器支持通过配置文件高度定制，并提供完整的单元测试覆盖。
